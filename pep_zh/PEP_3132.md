# PEP 3132 – 扩展的可迭代对象解包

**Metadata:**

- **Author**: Georg Brandl <georg&#32;&#97;t&#32;python.org>
- **Created**: 30-Apr-2007
- **Python-Version**: 3.0

## 摘要

本 PEP 提议对可迭代对象解包语法进行改进，允许指定一个“通配”名称，该名称将被分配一个包含所有未分配给“常规”名称的元素的列表。

例子胜过千言万语：

```python
>>> a, *b, c = range(5)
>>> a
0
>>> c
4
>>> b
[1, 2, 3]
```

## 理由

许多算法需要将序列拆分为“首元素、其余元素”对。使用新语法，

```python
first, rest = seq[0], seq[1:]
```

可以被更简洁且可能更高效的方式替代：

```python
first, *rest = seq
```

对于更复杂的解包模式，新语法看起来更简洁，并且不再需要笨拙的索引处理。

此外，如果右侧的值不是列表而是可迭代对象，则必须先将其转换为列表才能进行切片操作；为了避免创建这个临时列表，人们不得不使用：

```python
it = iter(seq)
first = it.next()
rest = list(it)
```

## 规范

在简单赋值（增强赋值未定义解包）的左侧，元组（或列表）中最多可以包含一个前面带有单个星号的表达式（此后称为“星号”表达式，而列表中的其他表达式称为“必需”）。这指定了一个子表达式，该子表达式将被分配一个列表，包含从正在解包的可迭代对象中未被任何必需表达式分配的所有元素，或者如果没有这样的元素则为空列表。

例如，如果 seq 是一个可切片的序列，当 seq 至少有 2 个元素时，以下所有赋值都是等价的：

```python
a, b, c = seq[0], list(seq[1:-1]), seq[-1]
a, *b, c = seq
[a, *b, c] = seq
```

如果可迭代对象不包含足够的元素来分配给所有必需表达式，则这是一个错误（与当前情况相同）。

将星号表达式用作单独的赋值目标也是一个错误，例如：

```python
*a = range(5)
```

但是，这是有效的语法：

```python
*a, = range(5)
```

注意，本提案也适用于隐式赋值上下文中的元组，例如在 for 语句中：

```python
for a, *b in [(1, 2, 3), (4, 5, 6, 7)]:
 print(b)
```

将输出：

```
[2, 3]
[5, 6, 7]
```

星号表达式仅允许作为赋值目标，在其他地方使用它们（当然，函数调用中的星号参数除外）都是错误。

## 实现

### 文法更改

此功能需要一条新的文法规则：

```p
star_expr: ['*'] expr
```

在这两条规则中，expr 被更改为 star_expr：

```
comparison: star_expr (comp_op star_expr)*
exprlist: star_expr (',' star_expr)* [',']
```

### 编译器的更改

添加了一个新的 ASDL 表达式类型 Starred，它表示一个星号表达式。注意，这里引入的星号表达式元素是通用的，以后可以在非赋值上下文中用于其他目的，例如 yield *iterable 提案。

编译器被更改为识别所有星号表达式无效的情况，并将其标记为语法错误。

添加了一个新的字节码指令 UNPACK_EX，其参数的低 8 位是星号目标之前的必需目标数量，高 8 位是星号目标之后的必需目标数量。对于不带星号表达式的序列解包，保留旧的 UNPACK_ITERABLE 操作码。

### 字节码解释器的更改

ceval.c 中的函数 unpack_iterable() 被更改以处理扩展解包，通过一个 argcntafter 参数。在 UNPACK_EX 情况下，该函数将执行以下操作：

- 收集星号之前的必需目标的所有元素

- 将可迭代对象中的所有剩余元素收集到一个列表中

- 从列表中弹出星号之后的必需目标的元素

- 将单个元素和调整大小的列表推送到堆栈上

可以添加对已知类型（如列表或元组）的可迭代对象解包的快捷方式。

当前实现可以在 SourceForge 补丁跟踪器 [SFPATCH] 中找到。它现在包含一个最小测试用例。

## 接受

在 python-3000 列表 [1] 上简短讨论后，Guido 以当前形式接受了本 PEP。讨论过的可能更改有：

- 只允许星号表达式作为 exprlist 中的最后一个元素。这将简化解包代码，并允许星号表达式被分配一个迭代器。这种行为被拒绝，因为它会太令人惊讶。

- 尝试让星号目标与源可迭代对象具有相同的类型，例如，在 a, *b = 'hello'中，b 将被赋值为字符串'ello'。这看起来不错，但无法与所有可迭代对象一致地实现。

- 使星号目标为元组而不是列表。这将与函数的 *args 一致，但会使结果的进一步处理更加困难。
