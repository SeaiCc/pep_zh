# PEP 274 – 字典推导式

**元数据:**

- **作者**: Barry Warsaw <barry&#32;&#97;t&#32;python.org>
- **创建日期**: 25-Oct-2001
- **Python 版本**: 2.7, 3.0
- **发布历史**: 29-Oct-2001

## 摘要

PEP 202 介绍了一种名为"列表推导式"的 Python 语法扩展。本 PEP 提议一种类似的语法扩展，称为"字典推导式"或简称"dict 推导式"。您可以以非常类似于列表推导式的方式使用字典推导式，只不过它们生成 Python 字典对象而不是列表对象。

## 决议

本 PEP 最初是为包含在 Python 2.3 中而编写的。在观察到其大部分好处都被生成器表达式与 dict() 构造函数所吸收后，它被撤销了。

然而，Python 2.7 和 3.0 引入了此确切功能，以及密切相关的集合推导式。2012-04-09，通过将其状态更新为已接受并更新 Python 版本字段，修改了 PEP 以反映这一现实。还删除了开放问题部分，因为这些已由当前实现长期解决。

## 提议的解决方案

字典推导式与列表推导式完全相同，只不过您使用花括号而不是方括号来组合表达式。此外，for 关键字之前的左侧部分表达键和值，用冒号分隔。此符号专门设计用于让您想起应用于字典的列表推导式。

## 理由

有时您将数据排列为长度 2 的序列，并且想要将其转换为字典。在 Python 2.2 中，dict() 构造函数接受一个参数，该参数是长度 2 的序列，用作 (键，值) 对来初始化新字典对象。

然而，从内存或性能角度来看，将某些数据转换为长度 2 的序列的行为可能不方便或效率低下。此外，对于某些常见操作，例如将事物列表转换为事物集以进行快速重复项删除或集包含测试，更好的语法可以帮助代码清晰度。

与列表推导式一样，始终可以使用显式的 for 循环（事实上，在早期版本的 Python 中，这是唯一的方法）。但与列表推导式一样，字典推导式可以提供比传统 for 循环更简洁的习语。

## 语义

字典推导式的语义实际上可以通过将列表推导式传递给内置字典构造函数在标准 Python 2.2 中演示：

```python
>>> dict([(i, chr(65+i)) for i in range(4)])
```

在语义上等同于：

```python
>>> {i : chr(65+i) for i in range(4)}
```

然而，字典构造函数方法有两个明显的缺点。首先，它不如字典推导式可读。其次，它强制程序员首先创建一个内存中的列表对象，这可能是昂贵的。

## 示例

```python
>>> print {i : chr(65+i) for i in range(4)}
{0 : 'A', 1 : 'B', 2 : 'C', 3 : 'D'}

>>> print {k : v for k, v in someDict.iteritems()} == someDict.copy()
1

>>> print {x.lower() : 1 for x in list_of_email_addrs}
{'barry@zope.com' : 1, 'barry@python.org' : 1, 'guido@python.org' : 1}

>>> def invert(d):
...     return {v : k for k, v in d.iteritems()}
...
>>> d = {0 : 'A', 1 : 'B', 2 : 'C', 3 : 'D'}
>>> print invert(d)
{'A' : 0, 'B' : 1, 'C' : 2, 'D' : 3}

>>> {(k, v): k+v for k in range(4) for v in range(4)}
... {(3, 3): 6, (3, 2): 5, (3, 1): 4, (0, 1): 1, (2, 1): 3,
     (0, 2): 2, (3, 0): 3, (0, 3): 3, (1, 1): 2, (1, 0): 1,
     (0, 0): 0, (1, 2): 3, (2, 0): 2, (1, 3): 4, (2, 2): 4, (
     2, 3): 5}
```
