# PEP 3148 – futures - 异步执行计算

**Metadata:**

- **Author**: Brian Quinlan <brian&#32;&#97;t&#32;sweetapp.com>
- **Created**: 16-Oct-2009
- **Python-Version**: 3.2

## Abstract

本 PEP 提出一个用于通过线程和进程评估可调用对象的包设计方案。

## Motivation

Python 目前拥有强大的原语来构建多线程和多进程应用程序，但并行化简单操作需要大量工作，例如显式启动进程/线程、构建工作/结果队列，并等待完成或其他终止条件（例如失败、超时）。此外，当每个组件都发明自己的并行执行策略时，也很难设计一个具有全局进程/线程限制的应用程序。

## Specification

### Naming

拟议的包将称为"futures"，并位于新的顶级包"concurrent"中。将 futures 库推入"concurrent"命名空间的理由有多个组成部分。第一个也是最简单的是防止与现有的"`from __future__ import x`"惯用法的任何混淆，该惯用法已在 Python 中使用了很长时间。此外，人们认为在名称前添加"concurrent"前缀完全表明了该库与什么相关——即并发——这应该消除任何额外的歧义，因为已经注意到社区中并非每个人都熟悉 Java Futures，或者 Futures 术语，除非它与美国股市有关。

最后，我们正在为标准库划分一个新的命名空间——显然命名为"concurrent"。我们希望将来向其中添加或移动现有的与并发相关的库。一个主要的例子是 multiprocessing.Pool 工作，以及该模块中包含的其他"addons"，它们跨线程和进程边界工作。

### Interface

拟议的包提供两个核心类：Executor 和 Future。Executor 接收异步工作请求（以可调用对象及其参数的形式），并返回一个 Future 来表示该工作请求的执行。

#### Executor

Executor 是一个抽象类，提供异步执行调用的方法。

`submit(fn, *args, **kwargs)`

> 安排可调用对象作为 fn(*args, **kwargs) 执行，并返回一个代表可调用对象执行的 Future 实例。这是一个抽象方法，必须由 Executor 子类实现。

`map(func, *iterables, timeout=None)`

> 等价于 map(func, *iterables)，但 func 是异步执行的，并且可能并发地调用 func 多次。返回的迭代器在调用__next__() 且在从原始 map() 调用后*timeout*秒内结果不可用时会引发 TimeoutError。如果未指定*timeout* 或为 None，则等待时间没有限制。如果调用引发异常，则在从迭代器检索其值时会引发该异常。

`shutdown(wait=True)`

> 向 executor 发出信号，当当前挂起的 futures 完成执行时释放其使用的任何资源。在 shutdown 之后对 Executor.submit 和 Executor.map 的调用将引发 RuntimeError。如果 wait 为 True，则此方法在所有挂起的 futures 完成执行并与 executor 关联的资源被释放之前不会返回。如果 wait 为 False，则此方法将立即返回，并且与 executor 关联的资源将在所有挂起的 futures 完成执行时被释放。无论 wait 的值如何，整个 Python 程序在所有挂起的 futures 完成执行之前都不会退出。

`__enter__()`
`__exit__(exc_type, exc_val, exc_tb)`

> 当将 executor 用作上下文管理器时，__exit__将调用 Executor.shutdown(wait=True)。

#### ProcessPoolExecutor

ProcessPoolExecutor 类是一个 Executor 子类，使用进程池异步执行调用。传递给 ProcessPoolExecutor.submit 的可调用对象和参数必须可以根据与 multiprocessing 模块相同的限制进行 pickle 序列化。

从提交给 ProcessPoolExecutor 的可调用对象内部调用 Executor 或 Future 方法将导致死锁。

`__init__(max_workers)`

> 使用最多*max_workers*个进程的池异步执行调用。如果*max_workers*为 None 或未给出，则将创建与机器处理器数量一样多的工作进程。

#### ThreadPoolExecutor

ThreadPoolExecutor 类是一个 Executor 子类，使用线程池异步执行调用。

当与 Future 关联的可调用对象等待另一个 Future 的结果时，可能会发生死锁。例如：

```python
import time
def wait_on_b():
    time.sleep(5)
    print(b.result())  # b 永远无法完成，因为它在等待 a。
    return 5

def wait_on_a():
    time.sleep(5)
    print(a.result())  # a 永远无法完成，因为它在等待 b。
    return 6

executor = ThreadPoolExecutor(max_workers=2)
a = executor.submit(wait_on_b)
b = executor.submit(wait_on_a)
```

以及：

```python
executor = ThreadPoolExecutor(max_workers=1)
executor.submit(wait_on_future)
def wait_on_future():
    f = executor.submit(pow, 5, 2)
    # 这将永远无法完成，因为只有一个工作线程并且
    # 它正在执行此函数。
    print(f.result())

executor = ThreadPoolExecutor(max_workers=1)
executor.submit(wait_on_future)
```

`__init__(max_workers)`

> 使用最多*max_workers*个线程的池异步执行调用。

#### Future Objects

Future 类封装了可调用对象的异步执行。Future 实例由 Executor.submit 返回。

`cancel()`

> 尝试取消调用。如果调用当前正在执行，则无法取消它，方法将返回 False，否则调用将被取消，方法将返回 True。

`cancelled()`

> 如果调用成功取消，则返回 True。

`running()`

> 如果调用当前正在执行且无法取消，则返回 True。

`done()`

> 如果调用成功取消或完成运行，则返回 True。

`result(timeout=None)`

> 返回调用返回的值。如果调用尚未完成，则此方法将等待最多*timeout*秒。如果调用在*timeout*秒内未完成，则将引发 TimeoutError。如果未指定*timeout* 或为 None，则等待时间没有限制。如果 future 在完成之前被取消，则将引发 CancelledError。

如果调用引发异常，则此方法将引发相同的异常。

`exception(timeout=None)`

> 返回调用引发的异常。如果调用尚未完成，则此方法将等待最多*timeout*秒。如果调用在*timeout*秒内未完成，则将引发 TimeoutError。如果未指定*timeout* 或为 None，则等待时间没有限制。如果 future 在完成之前被取消，则将引发 CancelledError。
如果调用完成而未引发异常，则返回 None。

`add_done_callback(fn)`

> 将一个可调用对象*fn*附加到 future，当 future 被取消或完成运行时将调用该可调用对象。*fn*将以 future 作为唯一参数被调用。添加的可调用对象按添加顺序调用，并且总是在添加它们的进程所属的线程中调用。如果可调用对象引发 Exception，则它将被记录并忽略。如果可调用对象引发另一个 BaseException，则行为未定义。
如果 future 已经完成或被取消，则 fn 将立即被调用。

##### Internal Future Methods

以下 Future 方法用于单元测试和 Executor 实现。

`set_running_or_notify_cancel()`

> 应由 Executor 实现在执行与 Future 关联的工作之前调用。如果此方法返回 False，则 Future 已被取消，即调用了 Future.cancel 并返回 True。任何等待 Future 完成（即通过 as_completed() 或 wait()）的线程都将被唤醒。
如果此方法返回 True，则 Future 未被取消并已进入运行状态，即对 Future.running() 的调用将返回 True。
此方法只能调用一次，并且不能在 Future.set_result() 或 Future.set_exception() 被调用后调用。

`set_result(result)`

> 设置与 Future 关联的工作的结果。

`set_exception(exception)`

> 将与 Future 关联的工作的结果设置为给定的 Exception。

#### Module Functions

`wait(fs, timeout=None, return_when=ALL_COMPLETED)`

> 等待由*fs*给出的 Future 实例（可能由不同的 Executor 实例创建）完成。返回一个名为的 2-元组集合。第一个集合，名为"done"，包含在等待完成之前完成（完成或被取消）的 futures。第二个集合，名为"not_done"，包含未完成 futures。timeout 可用于控制在返回之前等待的最大秒数。如果未指定 timeout 或为 None，则等待时间没有限制。

return_when 指示方法何时应该返回。它必须是以下常量之一：

| Constant | Description |
|----------|-------------|
| FIRST_COMPLETED | 当任何 future 完成或被取消时，方法将返回。 |
| FIRST_EXCEPTION | 当任何 future 通过引发异常完成时，方法将返回。如果没有 future 引发异常，则它等价于 ALL_COMPLETED。 |
| ALL_COMPLETED | 当所有调用完成时，方法将返回。 |

`as_completed(fs, timeout=None)`

> 返回一个迭代器，迭代由*fs*给出的 Future 实例，当 futures 完成（完成或被取消）时产生它们。任何在调用 as_completed() 之前完成的 futures 将首先被产生。返回的迭代器在调用__next__() 且在从原始 as_completed() 调用后*timeout*秒内结果不可用时会引发 TimeoutError。如果未指定*timeout* 或为 None，则等待时间没有限制。Future 实例可以由不同的 Executor 实例创建。

### Check Prime Example

```python
from concurrent import futures
import math

PRIMES = [
    112272535095293,
    112582705942171,
    112272535095293,
    115280095190773,
    115797848077099,
    1099726899285419]

def is_prime(n):
    if n % 2 == 0:
        return False

    sqrt_n = int(math.floor(math.sqrt(n)))
    for i in range(3, sqrt_n + 1, 2):
        if n % i == 0:
            return False
    return True

def main():
    with futures.ProcessPoolExecutor() as executor:
        for number, prime in zip(PRIMES, executor.map(is_prime,
                                                      PRIMES)):
            print('%d is prime: %s' % (number, prime))

if __name__ == '__main__':
    main()
```

### Web Crawl Example

```python
from concurrent import futures
import urllib.request

URLS = ['http://www.foxnews.com/',
        'http://www.cnn.com/',
        'http://europe.wsj.com/',
        'http://www.bbc.co.uk/',
        'http://some-made-up-domain.com/']

def load_url(url, timeout):
    return urllib.request.urlopen(url, timeout=timeout).read()

def main():
    with futures.ThreadPoolExecutor(max_workers=5) as executor:
        future_to_url = dict(
            (executor.submit(load_url, url, 60), url)
             for url in URLS)

        for future in futures.as_completed(future_to_url):
            url = future_to_url[future]
            try:
                print('%r page is %d bytes' % (
                          url, len(future.result())))
            except Exception as e:
                print('%r generated an exception: %s' % (
                          url, e))

if __name__ == '__main__':
    main()
```

## Rationale

本模块的拟议设计深受 Java java.util.concurrent 包 [1] 的影响。该模块的概念基础，与 Java 一样，是 Future 类，它表示异步计算的进度和结果。Future 类对所使用的评估模式几乎没有承诺，例如，它可以用于表示惰性或急切评估，用于使用线程、进程或远程过程调用进行评估。

Futures 由 Executor 类的具体实现（在 Java 中称为 ExecutorService）创建。参考实现提供了使用进程池或线程池来急切评估计算的类。

Futures 已经在 Python 中作为流行的 Python 食谱 [2] 的一部分被看到，并在 Python-3000 邮件列表 [3] 上讨论过。

拟议的设计是显式的，即它要求客户端意识到他们正在消耗 Futures。可以设计一个返回代理对象（类似于 weakref 风格）的模块，可以透明地使用它们。可以基于拟议的显式机制构建代理实现。

拟议的设计没有对 Python 语言语法或语义引入任何更改。可以引入特殊语法 [4] 来标记函数和方法调用为异步。在操作急切异步评估的同时将返回代理结果，并且仅当在操作完成之前使用代理对象时执行才会阻塞。

Anh Hai Trinh 提出了一个更简单但功能更有限的 API 概念 [5]，并且 API 已经在 stdlib-sig [6] 上进行了详细讨论。

拟议的设计在 Python-Dev 邮件列表 [7] 上进行了讨论。在这些讨论之后，进行了以下更改：

- Executor 类被制作为一个抽象基类

- 由于缺乏令人信服的使用案例，Future.remove_done_callback 方法被移除

- Future.add_done_callback 方法被修改以允许将相同的可调用对象添加多次

- Future 类的突变方法得到了更好的文档说明，以表明它们对创建它们的 Executor 是私有的
