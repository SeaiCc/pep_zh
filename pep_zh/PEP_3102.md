# PEP 3102 – 仅限关键字的参数

**Metadata:**

- **Author**: Talin <viridia&#32;&#97;t&#32;gmail.com>
- **Created**: 22-Apr-2006
- **Python-Version**: 3.0
- **Post-History**: 28-Apr-2006, 19-May-2006

## 摘要

本 PEP 提议对函数参数赋值到命名参数槽的方式进行更改。特别是，它允许声明“仅限关键字”的参数：只能通过关键字提供的参数，这些参数永远不会被位置参数自动填充。

## 理由

当前 Python 函数调用范式允许通过位置或关键字来指定参数。参数可以显式地通过名称填充，也可以隐式地通过位置填充。

很多时候，人们希望函数能够接受可变数量的参数。Python 语言使用'varargs'语法（*name）来支持这一点，它指定任何“剩余”的参数作为元组传递给 varargs 参数。

这的一个限制是，目前，所有的常规参数槽必须在 vararg 槽之前被填充。

这并不总是理想的。人们很容易想象一个函数接受可变数量的参数，但也接受一个或多个以关键字参数形式的“选项”。目前，实现这一点的唯一方法是同时定义一个 varargs 参数和一个'keywords'参数（**kwargs），然后手动从字典中提取所需的关键字。

## 规范

在语法上，提议的更改相当简单。第一个更改是允许常规参数出现在 varargs 参数之后：

```python
def sortwords(*wordlist, case_sensitive=False):
 ...
```

此函数接受任意数量的位置参数，并且它还接受一个名为'case_sensitive'的关键字选项。这个选项永远不会被位置参数填充，但必须通过名称显式指定。

仅限关键字的参数不一定要有默认值。由于 Python 要求所有参数都必须绑定到一个值，而将值绑定到仅限关键字参数的唯一方法是通过关键字，因此这样的参数是“必需关键字”参数。这样的参数必须由调用者提供，并且必须通过关键字提供。

第二个语法更改是允许 varargs 参数省略参数名称。这样做的含义是允许那些本来不需要 varargs 参数的函数使用仅限关键字的参数：

```python
def compare(a, b, *, key=None):
 ...
```

这个更改背后的推理如下。想象一个函数，它接受几个位置参数以及一个关键字参数：
```python
def compare(a, b, key=None):
 ...
```

现在，假设你想要让'key'成为一个仅限关键字的参数。在上述语法下，你可以通过在关键字参数之前立即添加一个 varargs 参数来实现：

```python
def compare(a, b, *ignore, key=None):
 ...
```

不幸的是，'ignore'参数也会吸收调用者可能提供的任何错误的位置参数。鉴于我们更希望任何不需要的参数引发错误，我们可以这样做：

```python
def compare(a, b, *ignore, key=None):
  if ignore:  # 如果 ignore 不为空
    raise TypeError
```

作为一个方便的快捷方式，我们可以简单地省略'ignore'名称，这意味着“不允许任何额外的位置参数”。

（注意：在对替代语法提案进行了大量讨论之后，BDFL 宣布支持这种用于指示位置参数结束的单星号语法。）

## 函数调用行为

前一节描述了旧行为和新行为之间的差异。然而，也有必要有一个独立的新行为描述，而不参考以前的模型。因此，下一节将尝试提供这样的描述。

当调用函数时，输入参数按以下方式分配给形式参数：

- 每个形式参数都有一个槽，用于包含分配给该参数的参数的值。

- 已经分配了值的槽被标记为“已填充”。尚未分配任何值的槽被视为“空”。

- 最初，所有槽都被标记为空。

- 首先分配位置参数，然后是关键字参数。

- 对于每个位置参数：
尝试将参数绑定到第一个未填充的参数槽。如果该槽不是 vararg 槽，则将该槽标记为“已填充”。

- 如果下一个未填充的槽是一个 vararg 槽，并且它没有名称，则这是一个错误。

- 否则，如果下一个未填充的槽是一个 vararg 槽，则所有剩余的 non-keyword 参数都被放入 vararg 槽中。

- 对于每个关键字参数：
如果有一个与关键字同名的参数，则参数值被分配给该参数槽。但是，如果参数槽已经填充，则这是一个错误。

- 否则，如果有一个'关键字字典'参数，则该参数被添加到字典中，使用关键字名称作为字典键，除非已经存在具有该键的条目，在这种情况下它是一个错误。

- 否则，如果没有关键字字典，也没有匹配的名称参数，则这是一个错误。

- 最后：
如果 vararg 槽尚未填充，则将空元组分配为其值。

- 对于每个剩余的空槽：如果该槽有默认值，则用默认值填充该槽。如果没有默认值，则这是一个错误。

根据当前的 Python 实现，遇到的任何错误都将通过引发 TypeError 来发出信号。（如果你想要不同的东西，那是另一个 PEP 的主题。）

## 向后兼容性

本 PEP 中指定的函数调用行为是现有行为的超集——也就是说，预计任何现有程序都将继续工作。
