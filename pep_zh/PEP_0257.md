# PEP 257 – 文档字符串规范

**元数据：**

- **作者**: David Goodger <goodger&#32;&#97;t&#32;python.org>, Guido van Rossum <guido&#32;&#97;t&#32;python.org>
- **创建时间**: 29-May-2001
- **发布历史**: 13-Jun-2001

## 摘要

本 PEP 记录了与 Python 文档字符串相关的语义和规范。

## 理由

本 PEP 的目标是标准化文档字符串的高级结构：它们应该包含什么，以及如何表述（不涉及文档字符串中的任何标记语法）。PEP 包含规范，而不是法律或语法。

> "通用规范提供了所有可维护性、清晰性、一致性，以及良好编程习惯的基础。它所没有做的是坚持要求你违背意愿地遵守它。这就是 Python！"——Tim Peters on comp.lang.python, 2001-06-16

如果您违反这些规范，最坏的情况是会受到一些不赞成的目光。但一些软件（如 Docutils 文档字符串处理系统 PEP 256、PEP 258）将了解这些规范，因此遵循它们会获得最佳结果。

## 规范

### 什么是文档字符串？

文档字符串是出现在模块、函数、类或方法定义中的第一个语句的字符串字面量。这样的文档字符串成为该对象的 `__doc__` 特殊属性。

所有模块通常都应该有文档字符串，模块导出的所有函数和类也应该有文档字符串。公共方法（包括 `__init__` 构造函数）也应该有文档字符串。包可以在包目录中 `__init__.py` 文件的模块文档字符串中记录。

Python 代码中其他位置出现的字符串字面量也可作为文档。它们不被 Python 字节码编译器识别，也不能作为运行时对象属性访问（即未分配给 `__doc__`），但软件工具可以提取两种类型的额外文档字符串：

- 出现在模块、类或 `__init__` 方法顶层简单赋值之后的字符串字面量称为"属性文档字符串"。

- 出现在另一个文档字符串之后的字符串字面量称为"附加文档字符串"。

有关属性和附加文档字符串的详细描述，请参阅 PEP 258，"Docutils 设计规范"。

为了保持一致性，始终在文档字符串周围使用"""三重双引号"""。如果您在文档字符串中使用任何反斜杠，请使用 r"""原始三重双引号"""。

文档字符串有两种形式：单行和多行文档字符串。

### 单行文档字符串

单行适用于非常明显的情况。它们应该真的适合一行。例如：

```python
def kos_root():
    """Return the pathname of the KOS root directory."""
    global _kos_root
    if _kos_root: return _kos_root
    ...
```

注意：

- 即使字符串适合一行，也使用三重引号。这使得以后扩展它变得容易。

- 闭合引号与开头的引号在同一行。这对单行看起来更好。

- 文档字符串之前或之后都没有空行。

- 文档字符串是一个以句号结尾的短语。它作为命令（"执行这个"，"返回那个"）描述函数或方法的效果，而不是作为描述；例如，不要写"Returns the pathname …"。

- 单行文档字符串不应是重申函数/方法参数的"签名"（可以通过内省获得）。不要这样做：

  ```python
  def function(a, b):
      """function(a, b) -> list"""
  ```

  这种类型的文档字符串仅适用于 C 函数（如内置函数），其中无法进行内省。然而，无法通过内省确定返回值，因此应该提到它。这种文档字符串的首选形式如下：

  ```python
  def function(a, b):
      """Do X and return a list."""
  ```

  （当然"Do X"应该替换为有用的描述！）

### 多行文档字符串

多行文档字符串由一个单行文档字符串的总结行组成，后跟一个空行，后跟更详细的描述。总结行可由自动索引工具使用；它适合一行并由空行与文档字符串的其余部分分开是很重要的。总结行可以与开头的引号在同一行或在下一行。整个文档字符串与其第一行的引号缩进相同（见下面的例子）。

在记录类的文档字符串（单行或多行）后插入一个空行——一般来说，类的方法由单个空行分隔，文档字符串需要通过空行与第一个方法分开。

脚本（独立程序）的文档字符串应可用作其"用法"消息，在脚本以不正确或缺失的参数（或可能带有"-h"选项，表示"帮助"）调用时打印。这样的文档字符串应记录脚本的函数和命令行语法、环境变量和文件。用法消息可以相当详细（几屏），应足以让新用户正确使用命令，以及对高级用户的完整快速参考所有选项和参数。

模块的文档字符串通常应列出模块导出的类、异常和函数（以及任何其他对象），每个对象的单行摘要。（这些摘要通常比对象文档字符串中的总结行提供较少的细节。）包的文档字符串（即包的 `__init__.py` 模块的文档字符串）还应列出包导出的模块和子包。

函数或方法的文档字符串应总结其行为并记录其参数、返回值、副作用、引发的异常以及调用时的限制（如果适用）。应指示可选参数。应记录关键字参数是否是接口的一部分。

类的文档字符串应总结其行为并列出公共方法和实例变量。如果类打算被子类化，并且有供子类使用的额外接口，则应单独列出此接口（在文档字符串中）。类构造函数应在其 `__init__` 方法的文档字符串中记录。各个方法应由其自己的文档字符串记录。

如果一个类继承另一个类，并且其行为主要从该类继承，则其文档字符串应提及这一点并总结差异。使用动词"override"（重写）表示子类方法替换超类方法而不调用超类方法；使用动词"extend"（扩展）表示子类方法调用超类方法（除了自己的行为之外）。

不要在运行文本中使用 Emacs 约定，即将函数或方法的参数用大写字母提及。Python 区分大小写，参数名称可用于关键字参数，因此文档字符串应记录正确的参数名称。最好将每个参数列在单独的一行上。例如：

```python
def complex(real=0.0, imag=0.0):
    """Form a complex number.

    
    :param real: the real part (default 0.0)
    :param imag: the imaginary part (default 0.0)
    """
    if imag == 0.0 and real == 0.0:
        return complex_zero
    ...
```

除非整个文档字符串适合一行，否则将闭合引号放在单独的一行上。这样，Emacs 的 fill-paragraph 命令可以用于它。

### 处理文档字符串缩进

文档字符串处理工具将从文档字符串的第二行及以后各行中剥离统一的缩进量，等于第一行之后所有非空行的最小缩进。文档字符串第一行中的任何缩进（即，直到第一个换行符）都不重要并被移除。文档字符串中后续行的相对缩进保留。应从文档字符串的开头和结尾删除空行。

由于代码比文字更精确，以下是算法的实现：

```python
def trim(docstring):
    if not docstring:
        return ''
    # Convert tabs to spaces (following the normal Python rules)
    # and split into a list of lines:
    lines = docstring.expandtabs().splitlines()
    # Determine minimum indentation (first line doesn't count):
    indent = sys.maxsize
    for line in lines[1:]:
        stripped = line.lstrip()
        if stripped:
            indent = min(indent, len(line) - len(stripped))
    # Remove indentation (first line is special):
    trimmed = [lines[0].strip()]
    if indent < sys.maxsize:
        for line in lines[1:]:
            trimmed.append(line[indent:].rstrip())
    # Strip off trailing and leading blank lines:
    while trimmed and not trimmed[-1]:
        trimmed.pop()
    while trimmed and not trimmed[0]:
        trimmed.pop(0)
    # Return a single string:
    return '\n'.join(trimmed)
```

此示例中的文档字符串包含两个换行符，因此长度为 3 行。第一行和最后一行是空白的：

```python
def foo():
    """
    This is the second line of the docstring.
    """
```

说明：

```python
>>> print repr(foo.__doc__)
'\n This is the second line of the docstring.\n '
>>> foo.__doc__.splitlines()
['', ' This is the second line of the docstring.', ' ']
>>> trim(foo.__doc__)
'This is the second line of the docstring.'
```

一旦修剪，这些文档字符串是等价的：

```python
def foo():
    """A multi-line
    docstring.
    """

def bar():
    """
    A multi-line
    docstring.
    """
```
