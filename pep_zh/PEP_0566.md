## 摘要

本 PEP 描述了 Python 包核心元数据规范的 1.2 版和 2.1 版之间的更改。1.2 版在 PEP 345 中指定。

它还更改了字段规范的规范源为 Core Metadata Specification 参考文档，其中包括字段名称及其语义和用法的具体内容。

## 字段

支持的元数据字段名称和语义的规范源是 Core Metadata Specification 文档。

标记为"(Multiple use)"的字段可以在单个 PKG-INFO 文件中指定多次。其他字段在 PKG-INFO 文件中只能出现一次。标记为"(optional)"的字段不是有效 PKG-INFO 文件所必需的；所有其他字段必须存在。

### 2.1 版新增

#### Description-Content-Type（可选）

一个字符串，声明分发包描述中使用的标记语法（如果有），以便工具可以智能地渲染描述。

历史上，像 PyPI 这样的工具假设包的描述是按 reStructuredText (reST) 格式化的，如果描述不是有效的 reST，则回退到纯文本。

此字段的引入使 PyPI 能够支持其他类型的标记语法，而不需要做出此假设。

此字段的完整规范在 Core Metadata Specification 中定义。

#### Provides-Extra（可选，多次使用）

一个包含可选功能名称的字符串。必须是有效的 Python 标识符。可用于使依赖项取决于是否已请求可选功能。

此字段的引入使包安装工具（如 pip）能够确定给定包提供了哪些 extras，以及包发布工具（如 twine）可以检查使用 extras 的环境标记的问题。

此字段的完整规范在 Core Metadata Specification 中定义。

### 2.1 版更改

#### Name

此字段格式的规范现在与 PEP 508 中定义的发行版名称规范相同。

#### Description

除了 Description 标头字段外，分发包的描述也可以在消息正文中提供（即，在标头之后的完全空行之后，不需要缩进或其他特殊格式）。

## 版本指定符

版本编号要求和指定版本之间比较的语义在 PEP 440 中定义。PEP 440 中定义的直接引用也可作为版本指定符的替代方案。

遵循 PEP 508，版本指定符不再需要在 Requires-Dist、Provides-Dist、Obsoletes-Dist 或 Requires-External 字段中用括号括起来，因此例如 `requests >= 2.8.1` 现在是有效的值。推荐格式是不使用括号，但解析元数据的工具也应能够处理带括号的版本指定符。此外，公共索引服务器可以禁止这些字段中的严格版本匹配子句或直接引用。

版本指定符的用法与 PEP 345 的其他方面不变。

## 环境标记

环境标记是可以附加在分号（";"）后的字段末尾的标记，用于添加有关执行环境的条件。

用于声明此类条件的环境标记格式在 PEP 508 的环境标记部分中定义。

环境标记的用法与 PEP 345 的其他方面不变。

## JSON 兼容元数据

可能需要在不允许多个重复键的数据结构中存储元数据，例如 JSON。

将元数据字段转换为此类数据结构的规范方法如下：

- 原始键值格式应使用 email.parser.HeaderParser 读取；

- 所有转换后的键应转换为小写。连字符应替换为下划线，但否则应保留所有其他字符；

- 标记为"(Multiple-use）"的任何字段的转换值应是一个包含给定键的所有原始值的单个列表；

- Keywords 字段应通过在逗号处分割原始值转换为列表；

- 如果存在消息正文，则应设置为 description 键的值。

- 结果应存储为字符串键字典。

## 与 PEP 345 的差异摘要

- Metadata-Version 现在是 2.1。

- 字段现在通过 Core Metadata Specification 指定。

- 添加了两个新字段：Description-Content-Type 和 Provides-Extra

- Name 字段的可接受值现在按 PEP 508 指定。

- 添加了转换为 JSON 兼容数据结构的规范方法。
