# PEP 3109 – Python 3000 中的异常抛出

**Metadata:**

- **Author**: Collin Winter <collinwinter&#32;&#97;t&#32;google.com>
- **Created**: 19-Jan-2006
- **Python-Version**: 3.0

## 摘要

本 PEP 介绍了对 Python 异常抛出机制的更改，旨在减少行噪声和语言的复杂性。

## 理由

Python 的指导原则之一是"应该有一种——而且最好只有一种——显而易见的方法来做某件事"。Python 2.x 的 raise 语句违反了这一原则，允许多种方式表达相同的想法。例如，以下语句是等价的：

```python
raise E, V

raise E(V)
```

raise 语句还有第三种形式，允许将任意回溯附加到异常 [1]：

```python
raise E, V, T
```

其中 T 是一个回溯。如 PEP 344 中所指定的，Python 3.x 中的异常对象将具有__traceback__属性，允许将三表达式 raise 语句翻译为：

```python
raise E, V, T
```

被翻译为

```python
e = E(V)
e.__traceback__ = T
raise e
```

使用这些翻译，我们可以将 raise 语句从四种形式减少到两种：

- raise（不带参数）用于在 except 套件中重新抛出活动异常。

- raise EXCEPTION 用于抛出新的异常。这种形式有两个子变体：EXCEPTION 可以是异常类或异常类的实例；有效的异常类是 BaseException 及其子类（PEP 352）。如果 EXCEPTION 是子类，它将不带参数被调用以获取异常实例。抛出其他任何内容都是错误。

通过这种整合，可以获得进一步、更具体的好处，正如 A.M. Kuchling 所指出的 [2]。

```python
PEP 8 没有表达对两种 raise 语句形式的偏好：
raise ValueError, 'blah'
raise ValueError("blah")

我更喜欢第二种形式，因为如果异常参数很长或包含字符串格式化，你不需要使用行连接字符，因为有包含的括号。
```

BDFL 已同意 [3] 并支持整合几种 raise 形式。

## 文法更改

在 Python 3 中，raise 语句的文法将从 [1]
```
raise_stmt: 'raise' [test [',' test [',' test]]]
```
更改为
```
raise_stmt: 'raise' [test]
```
## 内置类型的更改

由于其与异常抛出的关系，生成器对象上 throw() 方法的签名将更改，删除可选的第二个和第三个参数。因此签名从（PEP 342）

```python
generator.throw(E, [V, [T]])
```

更改为
```python
generator.throw(EXCEPTION)
```
其中 EXCEPTION 是 BaseException 的子类或 BaseException 子类的实例。

## 语义更改

在 Python 2 中，以下 raise 语句是合法的
```
raise ((E1, (E2, E3)), E4), V
```
解释器将采用元组的第一个元素作为异常类型（递归地），使上述完全等价于
```
raise E1, V
```
从 Python 3.0 开始，将不再支持抛出这样的元组。此更改将使 raise 语句与生成器对象上的 throw() 方法保持一致，后者已经不允许这样做。

## 兼容性问题

所有二表达式和三表达式的 raise 语句都需要修改，正如所有二表达式和三表达式的生成器 throw() 调用一样。幸运的是，在这种情况下，从 Python 2.x 到 Python 3.x 的转换很简单，可以通过 Guido van Rossum 的 2to3 工具 [4] 使用 raise 和 throw fixers（[5]、[6]）机械地处理。

将执行以下转换：

- 零表达式和一表达式的 raise 语句将保持不变。

- 二表达式的 raise 语句将从 raise E, V

转换为
```
raise E(V)
```
二表达式的 throw() 调用将从
```
generator.throw(E, V)
```
转换为
```
generator.throw(E(V))
```
参见第 5 点以了解此转换的注意事项。

- 三表达式的 raise 语句将从 `raise E, V, T`

转换为
```python
e = E(V)
e.__traceback__ = T
raise e
```
三表达式的 throw() 调用将从
```
generator.throw(E, V, T)
```
转换为
```
e = E(V)
e.__traceback__ = T
generator.throw(e)
```
参见第 5 点以了解此转换的注意事项。

- 二表达式和三表达式的 raise 语句，其中 E 是元组文字，可以使用 2to3 的 raise fixer 自动转换。E 是非文字元组的 raise 语句，例如函数调用的结果，需要手动转换。

- 二表达式和三表达式的 raise 语句，其中 E 是异常类而 V 是异常实例，需要特别关注。这些情况分为两类：raise E, V 作为零参数 raise 语句的长写法。例如，假设 F 是 E 的子类：
```python
try:
 something()
except F as V:
 raise F(V)
except E as V:
 handle(V)
```

这可以更好地表达为
```python
try:
 something()
except F:
 raise
except E as V:
 handle(V)
```
- `raise E, V` 作为将异常"转换"为另一个类的方法。从 `distutils.compiler.unixcompiler` 中取一个例子：
```python
try:
 self.spawn(pp_args)
except DistutilsExecError as msg:
 raise CompileError(msg)
```

这可以更好地表达为

```python
try:
 self.spawn(pp_args)
except DistutilsExecError as msg:
 raise CompileError from msg
```

使用 PEP 344 中引入的 `raise ... from ...` 语法。
