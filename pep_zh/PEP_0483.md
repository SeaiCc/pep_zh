## 摘要

本 PEP 阐述了 PEP 484 引用的理论。

## 介绍

本文档阐述了 Python 3.5 新类型提示提案的理论。它还不是完整的提案或规范，因为还有许多细节需要解决，但它阐述了理论，没有它很难讨论更详细的规范。我们首先回顾类型论的基本概念；然后解释渐进式类型；接着陈述一些通用规则并定义可以在注解中使用的新特殊类型（如 Union）；最后定义泛型类型和类型提示的实用方面的方法。

### 符号约定

- t1、t2 等和 u1、u2 等是类型。有时我们写 ti 或 tj 表示"t1、t2 等中的任意一个"

- T、U 等是类型变量（使用 TypeVar() 定义，见下文）。

- 对象、使用 class 语句定义的类和实例使用标准 PEP 8 约定表示。

- 应用于 PEP 上下文中类型的符号 == 表示两个表达式表示相同的类型。

- 注意 PEP 484 区分了类型和类（类型是类型检查器的概念，而类是运行时概念）。在本 PEP 中，我们澄清这个区别，但避免不必要的严格性，以允许类型检查器实现更多的灵活性。

## 背景

文献中有许多关于类型概念的定义。在这里，我们假设类型是一组值和一组可以应用于这些值的函数。

有几种方法可以定义特定类型：

- 通过明确列出所有值。例如，True 和 False 形成类型 bool。

- 通过指定可以与该类型变量一起使用的函数。例如，所有具有__len__方法的对象形成类型 Sized。[1, 2, 3] 和'abc'都属于此类型，因为可以对它们调用 len：len([1, 2, 3]) # OK
len('abc') # also OK
len(42) # not a member of Sized

- 通过简单的类定义，例如，如果定义一个类：class UserID(int):
    pass

那么该类的所有实例也形成类型。

- 还有更复杂的类型。例如，可以定义类型 FancyList 为所有仅包含 int、str 实例或其子类的列表。值 [1, 'abc', UserID(42)] 有这个类型。

用户能够以类型检查器可以理解的形式定义类型是很重要的。本 PEP 的目标是提出这样一种系统化的类型定义方法，用于使用 PEP 3107 语法的变量和函数的类型注解。这些注解可用于避免多种类型的错误，用于文档目的，或者甚至可能提高程序执行速度。在这里，我们只关注通过使用静态类型检查器来避免错误。

### 子类型关系

静态类型检查器的一个关键概念是子类型关系。它源于这个问题：如果 first_var 具有类型 first_type，而 second_var 具有类型 second_type，赋值 first_var = second_var 是否安全？

何时安全的一个强标准是：

- second_type 的每个值也在 first_type 的值集中；并且

- first_type 的每个函数也在 second_type 的函数集中。

因此定义的关系称为子类型关系。

根据此定义：

- 每个类型都是它自己的子类型。

- 值集在子类型化过程中变小，而函数集变大。

一个直观的例子：每只 Dog 都是 Animal，同时 Dog 有更多功能，例如它可以吠叫，因此 Dog 是 Animal 的子类型。相反，Animal 不是 Dog 的子类型。

一个更正式的示例：整数是实数的子类型。确实，每个整数当然也是一个实数，并且整数支持更多操作，例如，按位移位 << 和 >>：

```python
lucky_number = 3.14 # type: float
lucky_number = 42 # Safe
lucky_number * 2 # This works
lucky_number << 5 # Fails

unlucky_number = 13 # type: int
unlucky_number << 5 # This works
unlucky_number = 2.72 # Unsafe
```

让我们考虑一个棘手的例子：如果 List[int] 表示类型由所有仅包含整数的列表组成，那么它不是 List[float] 的子类型，由所有仅包含实数的列表组成。子类型化的第一个条件成立，但追加实数仅在 List[float] 中有效，因此第二个条件失败：

```python
def append_pi(lst: List[float]) -> None:
    lst += [3.14]

my_list = [1, 3, 5] # type: List[int]

append_pi(my_list) # Naively, this should be safe...

my_list[-1] << 5 # ... but this fails
```

有两种广泛的方法来向类型检查器声明子类型信息。

在名义子类型化中，类型树基于类树，即，UserID 被认为是 int 的子类型。此方法应在类型检查器的控制下使用，因为在 Python 中可以用不兼容的方式覆盖属性：

```python
class Base:
    answer = '42' # type: str

class Derived(Base):
    answer = 5 # should be marked as error by type checker
```

在结构子类型化中，子类型关系是从声明的方法推导出来的，即，UserID 和 int 将被视为相同类型。虽然这可能会偶尔造成混淆，结构子类型化被认为更灵活。我们努力同时支持这两种方法，以便结构信息可用于名义子类型化之外。

## 渐进式类型概述

渐进式类型允许仅标注程序的一部分，从而利用动态类型和静态类型的理想方面。

我们定义了一个新关系 is-consistent-with，它类似于 is-subtype-of，除了当新类型 Any 涉及时它不是传递的。（两个关系都不是对称的。）分配 a_value 到 a_variable 是可以的，如果 a_value 的类型与 a_variable 的类型一致。（将此与"...如果 a_value 的类型是 a_variable 类型的子类型"进行比较，它陈述了面向对象编程的基本原理之一。）is-consistent-with 关系由三个规则定义：

- 如果 t1 是 t2 的子类型，则类型 t1 与类型 t2 一致。（但不是反过来。）

- Any 与每个类型一致。（但 Any 不是每个类型的子类型。）

- 每个类型与 Any 一致。（但每个类型都不是 Any 的子类型。）

就是这样！见 Jeremy Siek 的博客文章 What is Gradual Typing 更详细的解释和动机。Any 可以被认为是一种类型具有所有值和方法。结合上面定义的类型化，这将 Any 部分放在类型层次结构的顶部（它具有所有值）和底部（它具有所有方法）。与此对比 object——它与大多数类型不一致（例如，你不能使用 object() 实例，其中 int 被期望）。IOW both Any 和 object 意思是"允许任何类型"当用于注解参数时，但只有 Any 可以被传递，无论期望什么类型（本质上，Any 声明回退到动态类型并停止抱怨来自静态检查器）。

下面是一个显示这些规则在实践中如何工作的示例：

假设我们有一个 Employee 类和一个子类 Manager：

```python
class Employee: ...
class Manager(Employee): ...
```

假设变量 worker 被声明为类型 Employee：

```python
worker = Employee() # type: Employee
```

现在可以将 Manager 实例分配给 worker（规则 1）：

```python
worker = Manager()
```

将 Employee 实例分配给声明为类型 Manager：

```python
boss = Manager() # type: Manager
boss = Employee() # Fails static check
```

但是，假设我们有一个类型为 Any 的变量：

```python
something = some_func() # type: Any
```

现在可以将 something 分配给 worker（规则 2）：

```python
worker = something # OK
```

当然，将 worker 分配给 something 也没问题（规则 3），但我们不需要一致性概念：

```python
something = worker # OK
```

### 类型与类

在 Python 中，类是由 class 语句定义的对象工厂，并由 type(obj) 内置函数返回。类是动态的、运行时概念。

类型概念如上所述，类型出现在变量和函数类型注解中，可以从下面描述的基本模块构建，并由静态类型检查器使用。

每个类都是一个类型，如上所述。但实现一个完全代表语义的给定类型是棘手的且容易出错，这不是 PEP 484 的目标。PEP 484 中描述的静态类型不应与运行时类混淆。示例：

- int 是一个类和一个类型。

- UserID 是一个类和一个类型。

- Union[str, int] 是一个类型，但不是适当的类：class MyUnion(Union[str, int]): ... # raises TypeError
Union[str, int]() # raises TypeError

Typing 接口使用类实现，即，在运行时它是可能评估，例如，Generic[T].__bases__。但为了强调类之间的区别和类型适用以下通用规则：

下面定义的类型（即 Any、Union 等）不能实例化，尝试这样做将引发 TypeError。（但 Generic 的非抽象子类可以。）

- 下面定义的类型不能被继承，除了 Generic 和从中导出的类。

- 所有这些如果在 isinstance 或 issubclass 中出现将引发 TypeError（未参数化的泛型除外）。

### 基本构建块

- Any。每个类型与 Any 一致；并且它也与每个类型一致（见上文）。

- Union[t1, t2, …]。至少一个是 t1 等的子类型的类型是此的子类型。所有组件都是 t1 等的子类型的联合是此的子类型。示例：Union[int, str] 是 Union[int, float, str] 的子类型。

- 参数的顺序无关紧要。示例：Union[int, str] == Union[str, int]。

- 如果 ti 本身是 Union，则结果被展平。示例：Union[int, Union[float, str]] == Union[int, float, str]。

- 如果 ti 和 tj 具有子类型关系，较不具体的类型存活。示例：Union[Employee, Manager] == Union[Employee]。

- Union[t1] 仅返回 t1。Union[] 是非法的，所以是 Union[()]

- 推论：Union[..., object, ...] 返回 object。

- Optional[t1]。Union[t1, None] 的别名，即 Union[t1, type(None)]。

- Tuple[t1, t2, …, tn]。项是 t1 实例的元组，等。示例：Tuple[int, float] 意味着两个项的元组，第一个是 int，第二个是 float；例如，(42, 3.14)。Tuple[u1, u2, ..., um] 是 Tuple[t1, t2, ..., tn] 的子类型如果它们具有相同的长度 n==m 且每个 ui 是 ti 的子类型。

- 要拼写空元组的类型，使用 Tuple[()]。

- 可变同质元组类型可以写成 Tuple[t1, ...]。（那是三个点，一个文字省略号；是的，那是 Python 语法中的有效令牌。）

- Callable[[t1, t2, …, tn], tr]。具有位置参数类型 t1 等，和返回类型 tr。参数列表可能是空 n==0。没有办法表示可选或关键字参数，nor varargs，但你可以说参数列表完全是通过编写 Callable[..., tr] 未检查（再次，文字省略号）。

我们可以添加：

- Intersection[t1, t2, …]。是 t1 等的子类型的类型是此的子类型。（与 Union 比较，它有 at least one 而不是 each 在其定义中。）参数的顺序无关紧要。嵌套交集被展平，例如 Intersection[int, Intersection[float, str]] == Intersection[int, float, str]。

- 较少类型的交集是交集的超类型更多类型，例如 Intersection[int, str] 是超类型 of Intersection[int, float, str]。

- 一个参数的交集就是那个参数，例如 Intersection[int] 是 int。

- 当参数具有子类型关系时，更具体的类型存活，例如 Intersection[str, Employee, Manager] 是 Intersection[str, Manager]。

- Intersection[] 是非法的，所以是 Intersection[()]。

- 推论：Any 从参数列表中消失，例如 Intersection[int, str, Any] == Intersection[int, str]。Intersection[Any, object] 是 object。

- Intersection 和 Union 之间的交互很复杂，但如果您理解交互，则不应该感到惊讶常规集的交集和并集（注意类型集可以是无限大小，因为没有任何限制新子类的数量）。

## 泛型类型

上面定义的基本构建块允许以通用方式构造新类型方式。例如，Tuple 可以采用具体类型 float 并创建具体类型 Vector = Tuple[float, ...]，或者它可以采用另一个类型 UserID 并创建另一个具体类型 Registry = Tuple[UserID, ...]。这种语义称为泛型类型构造函数，它类似于函数的语义，但是函数采用值并返回值，而泛型类型构造函数采用类型并"返回"一个类型。

当特定类或函数表现得如此类型泛型方式时，这是很常见的。考虑两个示例：

- 容器类，如 list 或 dict，通常仅包含特定类型的值。因此，用户可能希望将类型注解它们：users = [] # type: List[UserID]
users.append(UserID(42)) # OK
users.append('Some guy') # Should be rejected by the type checker

examples = {} # type: Dict[str, Any]
examples['first example'] = object() # OK
examples[2] = None # rejected by the type checker

以下函数可以采用两个类型为 int 的参数并返回一个 int，或采用两个类型为 float 的参数并返回一个 float 等：def add(x, y):
    return x + y

add(1, 2) == 3
add('1', '2') == '12'
add(2.7, 3.5) == 6.2

为了允许来自第一个示例的情况中的类型注解，内置容器和容器抽象基类扩展了类型参数，以便它们表现得像泛型类型构造函数。表现得像泛型类型构造函数的类称为泛型类型。示例：

```python
from typing import Iterable

class Task:
    ...

def work(todo_list: Iterable[Task]) -> None:
    ...
```

这里 Iterable 是一个泛型类型，采用具体类型 Task 并返回具体类型 Iterable[Task]。

表现得像泛型方式的函数（如第二个示例）称为泛型函数。泛型函数的类型注解允许使用类型变量。它们关于泛型类型的语义有些类似函数中参数的语义。但是人们不分配具体类型到类型变量，这是静态类型检查器的任务查找它们的可能的值，并在找不到时警告用户。示例：

```python
def take_first(seq: Sequence[T]) -> T: # a generic function
    return seq[0]

accumulator = 0 # type: int

accumulator += take_first([1, 2, 3]) # Safe, T deduced to be int
accumulator += take_first((2.7, 3.5)) # Unsafe
```

类型变量广泛用于类型注解，内部类型检查器中的类型推断机制通常构建在类型变量。因此，让我们详细考虑它们。

### 类型变量

X = TypeVar('X') 声明一个唯一的类型变量。名称必须匹配变量名。默认情况下，类型变量范围所有可能的类型。示例：

```python
def do_nothing(one_arg: T, other_arg: T) -> None:
    pass

do_nothing(1, 2) # OK, T is int
do_nothing('abc', UserID(42)) # also OK, T is object
```

Y = TypeVar('Y', t1, t2, ...)。同上，约束为 t1 等。行为类似于 Union[t1, t2, ...]。约束类型变量范围仅 over constrains t1 等。精确；约束的子类是被 t1 中最派生的基类替换，等。示例：

具有约束类型变量的函数类型注解：AnyStr = TypeVar('AnyStr', str, bytes)

```python
def longest(first: AnyStr, second: AnyStr) -> AnyStr:
    return first if len(first) >= len(second) else second

result = longest('a', 'abc') # The inferred type for result is str

result = longest('a', b'abc') # Fails static type check
```

在此示例中，longest() 的两个参数必须具有相同的类型（str 或 bytes），而且，即使参数是实例公共 str 子类，返回类型仍然是 str，而不是那个子类（见下一个示例）。

- 相比之下，如果类型变量未约束，公共子类将被选为返回类型，例如：S = TypeVar('S')

```python
def longest(first: S, second: S) -> S:
    return first if len(first) >= len(second) else second

class MyStr(str): ...

result = longest(MyStr('a'), MyStr('abc'))
```

result 的推断类型是 MyStr（而在 AnyStr 示例中 it would be str）。

- 同样用于比较，如果使用 Union，返回类型也必须是 a Union：U = Union[str, bytes]

```python
def longest(first: U, second: U) -> U:
    return first if len(first) >= len(second) else second

result = longest('a', 'abc')
```

result 的推断类型仍然是 Union[str, bytes]，即使两个参数都是 str。

注意类型检查器将拒绝此函数：

```python
def concat(first: U, second: U) -> U:
    return first + second # Error: can't concatenate str and bytes
```

对于参数只能同时更改其类型的情况应使用约束类型变量。

### 定义和使用泛型类型

用户可以使用声明他们的类为泛型类型特殊构建块 Generic。定义 class MyGeneric(Generic[X, Y, ...]): ... 定义泛型类型 MyGeneric 在类型变量 X 等上。MyGeneric 本身成为可参数化，例如 MyGeneric[int, str, ...] 是一个特定类型，具有替换 X -> int 等。示例：

```python
class CustomQueue(Generic[T]):

    def put(self, task: T) -> None:
        ...
    def get(self) -> T:
        ...

def communicate(queue: CustomQueue[str]) -> Optional[str]:
    ...
```

从泛型类型派生的类变得通用。类可以继承多个泛型类型。但是，从泛型返回的特定类型派生的类不是通用。示例：

```python
class TodoList(Iterable[T], Container[T]):
    def check(self, item: T) -> None:
        ...

def check_all(todo: TodoList[T]) -> None: # TodoList is generic
    ...

class URLList(Iterable[bytes]):
    def scrape_all(self) -> None:
        ...

def search(urls: URLList) -> Optional[bytes] # URLList is not generic
    ...
```

继承泛型类型对相应的施加子类型关系具体类型，以便 TodoList[t1] 是子类型 Iterable[t1] 在上面的例子中。

泛型类型可以在几步中专业化（索引）。每个类型变量都可以替换为特定类型或另一个泛型类型。如果 Generic 出现在基类列表中，那么它应该包含所有类型变量，并且类型参数的顺序是由它们在 Generic 中出现的顺序确定。示例：

```python
Table = Dict[int, T] # Table is generic
Messages = Table[bytes] # Same as Dict[int, bytes]

class BaseGeneric(Generic[T, S]):
    ...

class DerivedGeneric(BaseGeneric[int, T]): # DerivedGeneric has one parameter
    ...

SpecificType = DerivedGeneric[int] # OK

class MyDictView(Generic[S, T, U], Iterable[Tuple[U, T]]):
    ...

Example = MyDictView[list, int, str] # S -> list, T -> int, U -> str
```

如果泛型类型出现在类型注解中，类型变量被省略，假定它是 Any。这种形式可以用作回退动态类型，并允许用于 issubclass 和 isinstance。实例中的所有类型信息在运行时被擦除。示例：

```python
def count(seq: Sequence) -> int: # Same as Sequence[Any]
    ...

class FrameworkBase(Generic[S, T]):
    ...

class UserClass:
    ...

issubclass(UserClass, FrameworkBase) # This is OK

class Node(Generic[T]):
    ...

IntNode = Node[int]
my_node = IntNode() # at runtime my_node.__class__ is Node
 # inferred static type of my_node is Node[int]
```

### 协变和逆变

如果 t2 是 t1 的子类型，则泛型类型构造函数 GenType 被称为：

- 协变，如果 GenType[t2] 是 GenType[t1] 的子类型所有这样的 t1 和 t2。

- 逆变，如果 GenType[t1] 是 GenType[t2] 的子类型所有这样的 t1 和 t2。

- 不变，如果以上都不是真的。

为了更好地理解这个定义，让我们做一个类比普通函数。假设我们有：

```python
def cov(x: float) -> float:
    return 2*x

def contra(x: float) -> float:
    return -x

def inv(x: float) -> float:
    return x*x
```

如果 x1 < x2，那么总是 cov(x1) < cov(x2)，和 contra(x2) < contra(x1)，而什么都不能说关于 inv。用 is-subtype-of 替换 <，函数与泛型类型构造函数我们得到协变、逆变的示例，和不变行为。现在让我们考虑实际示例：

- Union 在其所有参数中表现得像协变。确实，如上所述，Union[t1, t2, ...] 是子类型 Union[u1, u2, ...]，如果 t1 是 u1 的子类型，等。

- FrozenSet[T] 也是协变的。让我们考虑 int 和 float 代替 T。首先，int 是 float 的子类型。其次，FrozenSet[int] 的值集是清楚地是 FrozenSet[float] 值的子集，而函数集 from FrozenSet[float] 是函数集的子集 from FrozenSet[int]。因此，根据定义 FrozenSet[int] 是 FrozenSet[float] 的子类型。

- List[T] 是不变的。确实，虽然 List[int] 的值集是 List[float] 值的子集，只有 int 可以追加 to a List[int]，如"Background"一节所述。因此，List[int] 不是 List[float] 的子类型。这是一个典型的可变类型的情况，它们通常是不变的。

说明（有些违反直觉）逆变行为的最好例子之一是可调用的类型。它在返回类型中是协变的，但是在参数。对于两个可调用的类型仅返回类型不同，子类型关系用于可调用的类型遵循返回类型。示例：

- Callable[[], int] 是 Callable[[], float] 的子类型。

- Callable[[], Manager] 是 Callable[[], Employee] 的子类型。

而对于两个仅不同的可调用类型只有一个参数的类型，子类型关系用于可调用类型的方向与参数类型。示例：

- Callable[[float], None] 是 Callable[[int], None] 的子类型。

- Callable[[Employee], None] 是 Callable[[Manager], None] 的子类型。

是的，你没看错。确实，如果可以计算经理薪水的函数被期望：

```python
def calculate_all(lst: List[Manager], salary: Callable[[Manager], Decimal]):
    ...
```

那么 Callable[[Employee], Decimal] 可以计算任何员工也是可以接受的。

Callable 的示例显示了如何制作更精确的类型注解对于函数：为每个参数选择最通用的类型，并为返回值选择最具体的类型。

可以通过声明用户定义的泛型类型的方差使用特殊关键字 covariant 和 contravariant 在用作参数的类型变量的定义。类型默认是不变的。示例：

```python
T = TypeVar('T')
T_co = TypeVar('T_co', covariant=True)
T_contra = TypeVar('T_contra', contravariant=True)

class LinkedList(Generic[T]): # invariant by default
    ...
    def append(self, element: T) -> None:
        ...

class Box(Generic[T_co]): # this type is declared covariant
    def __init__(self, content: T_co) -> None:
        self._content = content
    def get_content(self) -> T_co:
        return self._content

class Sink(Generic[T_contra]): # this type is declared contravariant
    def send_to_nowhere(self, data: T_contra) -> None:
        with open(os.devnull, 'w') as devnull:
            print(data, file=devnull)
```

注意，虽然方差是通过类型变量定义的，但它不是类型变量的属性，而是泛型类型的属性。在派生泛型的复杂定义中，方差仅从使用的类型变量确定。一个复杂的示例：

```python
T_co = TypeVar('T_co', Employee, Manager, covariant=True)
T_contra = TypeVar('T_contra', Employee, Manager, contravariant=True)

class Base(Generic[T_contra]):
    ...

class Derived(Base[T_co]):
    ...
```

类型检查器从第二个声明中找到 Derived[Manager] 是 Derived[Employee] 的子类型，和 Derived[t1] 是 Base[t1] 的子类型。如果我们用 < 表示 is-subtype-of 关系，那么此情况的完整子类型化图将是：

```
Base[Manager] > Base[Employee]
 v     v
Derived[Manager] < Derived[Employee]
```

因此，类型检查器还将找到，例如，Derived[Manager] 是 Base[Employee] 的子类型。

有关类型变量、泛型类型和方差的更多信息，见 PEP 484、mypy docs on generics, 和 Wikipedia。

## 实用主义

有些事情与理论无关，但使实际使用更加方便。（这不是完整的列表；我可能漏掉了一些，有些仍有争议或未完全指定。）

- 在期望类型的地方，None 可以替换为 type(None)；例如 Union[t1, None] == Union[t1, type(None)]。

- 类型别名，例如：Point = Tuple[float, float]
def distance(point: Point) -> float: ...

通过字符串的前向引用，例如：class MyComparable:
    def compare(self, other: 'MyComparable') -> int: ...

类型变量可以声明为未约束、约束、或无边界形式。泛型类型的方差也可以使用带有特殊关键字的类型变量声明参数，从而避免任何特殊语法，例如：T = TypeVar('T', bound=complex)

```python
def add(x: T, y: T) -> T:
    return x + y

T_co = TypeVar('T_co', covariant=True)

class ImmutableList(Generic[T_co]): ...
```

注释中的类型声明，例如：lst = [] # type: Sequence[int]

使用 cast(T, obj) 的转换，例如：zork = cast(Any, frobozz())

其他事情，例如重载和存根模块，见 PEP 484。
