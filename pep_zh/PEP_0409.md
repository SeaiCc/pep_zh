# PEP 409 – 抑制异常上下文

**元数据:**

- **作者**: Ethan Furman <ethan&#32;&#97;t&#32;stoneleaf.us>
- **创建日期**: 26-Jan-2012
- **Python 版本**: 3.3
- **发布历史**: 30-Aug-2002, 01-Feb-2012, 03-Feb-2012

## 摘要

PEP 3134 的未决问题之一是抑制上下文：目前无法做到这一点。本 PEP 提议一种方法。

## 理由

生成异常有两种基本方式：

- Python 生成（有 bug 的代码、缺失的资源、循环结束等）

- 手动生成（使用 raise 语句）

在编写库甚至自定义类时，可能会需要抛出异常；此外，从一种异常转换为另一种异常可能是有用的，甚至是必要的。以我的 dbf 模块为例：

```python
try:
    value = int(value)
except Exception:
    raise DbfError(...)
```

原始异常是什么（ValueError、TypeError 或其他）都无关紧要。从这一点开始，异常是 DbfError，原始异常没有价值。但是，如果打印此异常，我们目前会看到两个异常。

## 替代方案

提出了几种可能性：

- `raise NewException() as` 重用 as 关键字；可能会令人困惑，因为我们并没有真正重新抛出原始异常

- `raise NewException() from None` 遵循现有语法，显式声明原始异常

- `exc = NewException(); exc.__context__ = None; raise exc` 前一种方法的非常冗长的方式

- `raise NewException.no_context(...)` 将上下文抑制作为类方法

上述所有选项都需要对核心进行更改。

## 提议

我提议采用第二个选项：

```python
raise NewException from None
```

它的优势在于使用现有的显式设置原因的模式：

```python
raise KeyError() from NameError()
```

但因为原因是 None，所以默认异常打印例程不会显示之前的上下文。

## 实现讨论

注意：在此 PEP 接受后，PEP 415 中提出并接受了更清晰的实现机制。有关 Python 3.3 中实际使用的实现的更多详细信息，请参阅该 PEP。

目前，`__context__` 和 `__cause__` 的默认值都是 None。为了支持 `raise ... from None`（这将把 `__cause__` 设置为 None），我们需要为 `__cause__` 设置不同的默认值。有人就如何在语言层面实现这一点提出了几种想法：

- 覆盖之前的异常信息（绕过问题并将 `__cause__` 保留为 None）。
  被拒绝，因为这可能会因错误消息不佳而严重妨碍调试。

- 在 `__cause__` 中使用一个布尔值：False 将是默认值，当使用 `from ...` 时会被显式链式异常或 None 替换。
  被拒绝，因为这鼓励对 `__cause__` 使用两种不同的对象类型，其中一种（布尔值）不允许具有所有可能的值（True 永远不会被使用）。

- 创建一个特殊的异常类，`__NoException__`。
  被拒绝，因为这可能会令人困惑，可能会被用户错误地抛出，并且不像 None、True 和 False 那样是真正唯一的值。

- 使用 Ellipsis 作为默认值（... 单例）。
  已接受。

省略号在英语中常用作占位符，当省略单词时使用。这在这里对我们有利，作为 `__cause__` 被省略的信号，因此在 `__context__` 中查找更多详细信息。

Ellipsis 不是异常，因此不能被抛出。

只有一个 Ellipsis，因此没有未使用的值。

错误信息没有被丢弃，因此即使默认代码不这样做，自定义代码也可以跟踪整个异常链。

## 语言细节

为了支持 `raise Exception from None`，`__context__` 将保持不变，但 `__cause__` 将以 Ellipsis 开头，当使用 `raise Exception from None` 方法时会变为 None。

| 形式 | __context__ | __cause__ |
|------|-------------|-----------|
| raise | None | Ellipsis |
| reraise | previous exception | Ellipsis |
| reraise from None \| ChainedException | previous exception | None \| explicitly chained exception |

默认异常打印例程将：

- 如果 `__cause__` 是 Ellipsis，则打印 `__context__`（如果有的话）。

- 如果 `__cause__` 是 None，则不打印 `__context__`。

- 如果 `__cause__` 是其他任何值，则打印 `__cause__`。

在后两种情况下，将停止跟踪异常链。

由于 `__cause__` 的默认值现在是 Ellipsis，而 `raise Exception from Cause` 只是以下内容的语法糖：

```python
_exc = NewException()
_exc.__cause__ = Cause()
raise _exc
```

因此，Ellipsis 和 None 现在都可以作为原因：

```python
raise Exception from Ellipsis
```
