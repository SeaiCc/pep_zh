# PEP 3101 – 高级字符串格式化

**元数据:**

- **作者**: Talin <viridia&#32;&#97;t&#32;gmail.com>
- **创建日期**: 16-Apr-2006
- **Python 版本**: 3.0
- **发布历史**: 28-Apr-2006, 06-May-2006, 10-Jun-2007, 14-Aug-2007, 14-Sep-2008

## 摘要

本 PEP 提议一个新的内置字符串格式化操作系统，旨在替代现有的'%'字符串格式化运算符。

## 理由

Python 目前提供两种字符串插值方法：

- 字符串的'%'运算符。[1]

- string.Template 模块。[2]

本 PEP 的主要范围涉及内置字符串格式化操作的提议（换句话说，内置字符串类型的方法）。

'%'运算符主要受限于它是一个二元运算符，因此最多只能接受两个参数。其中一个参数已经专用于格式字符串，留下所有其他变量挤入剩余的参数中。当前的做法是使用字典或元组作为第二个参数，但正如许多人评论的那样 [3]，这缺乏灵活性。"全有或全无"的方法（意味着必须选择仅位置参数或仅命名参数）被认为过于限制。

虽然本提议与 string.Template 之间有一些重叠，但认为每个都服务于不同的需求，一个不会使另一个变得多余。本提议的机制像'%'一样，对于只使用一次的小字符串是有效的，因此，例如，本提议不考虑将字符串编译为模板，尽管本提议确实小心地定义格式字符串和 API，以便有效的模板包可以重用语法甚至一些底层格式化代码。

## 规范

规范将包括以下部分：

- 添加到内置字符串类的新格式化方法的规范。

- 添加到 string 模块的函数和标志值的规范，以便底层格式化引擎可以与额外选项一起使用。

- 格式字符串的新语法规范。

- 控制对象格式化和转换的新特殊方法集规范。

- 用户定义格式化类的 API 规范。

- 格式化错误如何处理的规范。

关于字符串编码的说明：在 Python 3.0 的背景下讨论此 PEP 时，假设所有字符串都是 unicode 字符串，并且在本文件背景下使用单词'string'通常指的是 Python 3.0 字符串，这与 Python 2.x unicode 对象相同。

在 Python 2.x 的背景下，本文档中使用单词'string'指的是可能是常规字符串或 unicode 对象的对象。本 PEP 中描述的所有函数调用接口都可以用于字符串和 unicode 对象，在所有情况下都有足够的信息能够正确推断输出字符串类型（换句话说，不需要两个单独的 API）。在所有情况下，格式字符串的类型占主导地位——也就是说，转换的结果将始终产生一个包含与输入格式字符串相同字符表示的对象。

### 字符串方法

内置字符串类（以及 2.6 中的 unicode 类）将获得一个新方法'format'，它接受任意数量的位置参数和关键字参数：

```python
"The story of {0}, {1}, and {c}".format(a, b, c=d)
```

在格式字符串内，每个位置参数用从零开始的数字标识，因此在上面的例子中，'a'是参数 0，'b'是参数 1。每个关键字参数由其关键字名称标识，因此在上面的例子中，'c'用于引用第三个参数。

还有一个全局内置函数'format'，它格式化单个值：

```python
print(format(10.0, "7.3g"))
```

此函数在后面的部分中描述。

### 格式字符串

格式字符串由交错的字符数据和标记组成。

字符数据是从格式字符串原封不动传输到输出字符串的数据；标记不是直接从格式字符串传输到输出，而是用于定义'替换字段'，这些字段描述格式引擎应该在输出字符串中放置什么来替代标记。

花括号字符（'curly braces'）用于指示字符串内的替换字段：

```python
"My name is {0}".format('Fred')
```

结果是字符串：

```
"My name is Fred"
```

花括号可以通过加倍来转义：

```python
"My name is {0} :-{{}}".format('Fred')
```

这将产生：

```
"My name is Fred :-{}"
```

花括号内的元素称为'字段'。字段由'字段名称'组成，可以是简单或复合的，以及可选的'格式说明符'。

### 简单和复合字段名称

简单字段名称是名称或数字。如果是数字，它们必须是有效的十进制整数；如果是名称，它们必须是有效的 Python 标识符。数字用于标识位置参数，而名称用于标识关键字参数。

复合字段名称是多个简单字段名称在表达式中的组合：

```python
"My name is {0.name}".format(open('out.txt', 'w'))
```

此示例显示了字段表达式中'getattr'或'点'运算符的使用。点运算符允许将输入值的属性指定为字段值。

与一些其他编程语言不同，您不能在格式字符串中嵌入任意表达式。这是故意设计的——您可以使用的表达式类型是故意限制的。只支持两个运算符：'.'（getattr）运算符和'[]'（getitem）运算符。允许这些运算符的原因是它们在非病态代码中通常不会产生副作用。

'getitem'语法的示例：

```python
"My name is {0[name]}".format(dict(name='Fred'))
```

应该注意的是，格式字符串中'getitem'的使用比其常规用法要有限得多。在上面的示例中，字符串'name'确实是字面字符串'name'，而不是名为'name'的变量。解析项键的规则非常简单。如果它以数字开头，则被视为数字，否则用作字符串。

因为键不是用引号分隔的，所以无法从格式字符串内指定任意字典键（例如，字符串"10"或":-]"）。

实现说明：本提议的实现不要求强制关于简单或点号名称是有效 Python 标识符的规则。相反，它将依赖底层对象的 getattr 函数在标识符不合法时抛出异常。str.format() 函数将有一个最小主义解析器，它只试图找出何时"完成"标识符（通过找到'.'或']'，或'}'等）。

### 格式说明符

每个字段还可以指定一组可选的'格式说明符'，用于调整该字段的格式。格式说明符跟随字段名称，用冒号（':'）字符分隔：

```python
"My name is {0:8}".format('Fred')
```

格式说明符的含义和语法取决于被格式化对象的类型，但对于不覆盖它们的任何对象，有一套标准的格式说明符。

格式说明符本身可以包含替换字段。例如，字段宽度本身是参数的字段可以指定为：

```python
"{0:{1}}".format(a, b)
```

这些'内部'替换字段只能出现在替换字段的格式说明符部分。内部替换字段本身不能有格式说明符。这也意味着替换字段不能嵌套到任意层级。

注意结尾的双'}'，通常会转义，在这种情况下没有转义。原因是 '{{'和'}}'转义语法仅用于格式字段外时应用。在格式字段内，花括号字符始终具有其正常含义。

格式说明符的语法是开放式的，因为类可以覆盖标准格式说明符。在这种情况下，str.format() 方法只是将所有字符从第一个冒号到匹配的花括号传递给相关的底层格式化方法。

### 标准格式说明符

如果对象未定义自己的格式说明符，则使用一组标准的格式说明符。这些在概念上与现有'%'运算符使用的格式说明符相似，但也有一些不同之处。

标准格式说明符的一般形式为：

```
[[fill]align][sign][#][0][minimumwidth][.precision][type]
```

括号 ([]) 表示可选元素。

然后可选的 align 标志可以是以下之一：

```
'<' - 强制字段在可用空间内左对齐（这是默认值。）
'>' - 强制字段在可用空间内右对齐。
'=' - 强制填充放置在符号（如果有）之后但在数字之前。这用于打印'+000000120'形式的字段。此对齐选项仅适用于数值类型。
'^' - 强制字段在可用空间内居中。
```

注意，除非定义了最小字段宽度，否则字段宽度将始终与填充它的数据大小相同，因此在这种情况下对齐选项没有意义。

可选的'fill'字符定义用于填充字段到最小宽度的字符。如果存在，填充字符必须后跟对齐标志。

'sign'选项仅适用于数值类型，可以是以下之一：

```python
'+' - 表示正数和负数都应使用符号
'-' - 表示仅对负数使用符号（这是默认行为）
' ' - 表示正数前应该使用前导空格
```

如果存在'#'字符，整数使用'替代形式'进行格式化。这意味着二进制、八进制和十六进制输出将分别以前缀'0b'、'0o'和'0x'开头。

'width'是一个定义最小字段宽度的十进制整数。如果未指定，则字段宽度将由内容决定。

如果宽度字段前面是零（'0'）字符，这将启用零填充。这等价于'='对齐类型和'0'填充字符。

'precision'是一个指示浮点转换中小数点后应显示多少位的十进制数。对于非数值类型，该字段指示最大字段大小——换句话说，将从字段内容中使用多少字符。精度在整数转换中被忽略。

最后，'type'确定数据应该如何呈现。

可用的整数呈现类型有：

```python
'b' - 二进制。以 2 为基数输出数字。
'c' - 字符。在打印之前将整数转换为相应的 Unicode 字符。
'd' - 十进制整数。以 10 为基数输出数字。
'o' - 八进制格式。以 8 为基数输出数字。
'x' - 十六进制格式。以 16 为基数输出数字，使用小写字母表示 9 以上的数字。
'X' - 十六进制格式。以 16 为基数输出数字，使用大写字母表示 9 以上的数字。
'n' - 数字。这与'd'相同， except 它使用当前区域设置插入适当的数字分隔符字符。
''（无）- 与'd'相同
```

可用的浮点呈现类型有：

```python
'e' - 指数表示法。使用字母'e'表示指数，以科学记数法打印数字。
'E' - 指数表示法。与'e'相同，except 它将数字转换为大写。
'f' - 定点。将数字显示为定点数。
'F' - 定点。与'f'相同，except 它将数字转换为大写。
'g' - 通用格式。这将数字打印为定点数，除非数字太大，在这种情况下它切换到'e'指数表示法。
'G' - 通用格式。与'g'相同，except 数字太大时切换到'E'。
'n' - 数字。这与'g'相同，except 它使用当前区域设置插入适当的数字分隔符字符。
'%' - 百分比。将数字乘以 100 并以定点（'f'）格式显示，后跟百分号。
''（无）- 类似于'g'，except 它至少打印小数点后一位数字。
```

对象能够定义自己的格式说明符以替换标准说明符。一个示例是'datetime'类，其格式说明符可能看起来像 strftime() 函数的参数：

```python
"Today is: {0:%a %b %d %H:%M:%S %Y}".format(datetime.now())
```

对于所有内置类型，空格式说明将产生相当于 str(value) 的结果。建议定义自己格式说明符的对象也遵循此约定。

### 显式转换标志

显式转换标志用于在格式化之前转换格式字段值。这可用于覆盖特定于类型的格式化行为，并将值格式化为更通用的类型。目前，识别两个显式转换标志：

```python
!r - 使用 repr() 将值转换为字符串。
!s - 使用 str() 将值转换为字符串。
```

这些标志放置在格式说明符之前：

```python
"{0!r:20}".format("Hello")
```

在上面的示例中，字符串"Hello"将带引号打印，在至少 20 个字符宽的字段中。

自定义 Formatter 类可以定义额外的转换标志。内置格式化器如果对无效的转换标志将引发 ValueError。

### 按类型控制格式化

每个 Python 类型可以通过定义__format__方法来控制其实例的格式化。__format__方法负责解释格式说明符、格式化值并返回结果字符串。

新的全局内置函数'format'简单地调用此特殊方法，类似于 len() 和 str() 简单地调用各自的特殊方法：

```python
def format(value, format_spec):
    return value.__format__(format_spec)
```

用"None"值调用此函数是安全的（因为 Python 中的"None"值是一个对象并且可以有方法。）

几个内置类型，包括'str'、'int'、'float'和'object'定义__format__方法。这意味着如果您从这些类型中的任何一个派生，您的类将知道如何格式化自己。

object.__format__方法是最简单的：它只是将对象转换为字符串，然后再次调用 format：

```python
class object:
    def __format__(self, format_spec):
        return format(str(self), format_spec)
```

'int'和'float'的__format__方法将根据格式说明符执行数值格式化。在某些情况下，这些格式化操作可以委托给其他类型。因此，例如，在'int'格式化器看到格式类型'f'（意味着'float'）的情况下，它可以简单地将值转换为 float 并再次调用 format()。

任何类都可以覆盖__format__方法以提供该类型的自定义格式化：

```python
class AST:
    def __format__(self, format_spec):
        ...
```

Python 2.x 说明：'format_spec'参数将是字符串对象或 unicode 对象，取决于原始格式字符串的类型。__format__方法应该测试 specifiers 参数的类型以确定是返回字符串还是 unicode 对象。__format__方法有责任返回适当类型的对象。

注意上面提到的'显式转换'标志不传递给__format__方法。相反，期望标志指定的转换在调用__format__之前执行。

### 用户定义的格式化

有时按类型自定义字段格式化是不够的。一个示例可能是电子表格应用程序，当值太大无法放入可用空间时显示井号'#'。

对于更强大和灵活的格式化，可以通过'string'模块中的'Formatter'类获取底层格式引擎的访问权限。此类接受正常 str.format 方法无法访问的额外选项。

应用程序可以子类化 Formatter 类来创建自己的自定义格式化行为。

PEP 不尝试确切地指定 Formatter 类定义的所有方法和属性；相反，这些将在初始实现中定义和记录。但是，本 PEP 将指定 Formatter 类的一般要求，如下所列。

虽然 string.format() 不直接使用 Formatter 类进行格式化，但两者使用相同的底层实现。string.format() 不直接使用 Formatter 类的原因是"string"是内置类型，这意味着它的所有方法必须在 C 中实现，而 Formatter 是 Python 类。Formatter 提供了围绕与 string.format() 使用的相同 C 函数的可扩展包装器。

### Formatter 方法

Formatter 类不接受初始化参数：

```python
fmt = Formatter()
```

类 Formatter 的公共 API 方法如下：

```
-- format(format_string, *args, **kwargs)
-- vformat(format_string, args, kwargs)
```

'format'是主要 API 方法。它接受格式模板和任意一组位置参数和关键字参数。'format'只是调用'vformat'的包装器。

'vformat'是执行实际格式化工作的函数。它作为单独的函数公开，用于您想要传入预定义参数字典的情况，而不是使用*args 和**kwds 语法解包和重新打包字典为单个参数。'vformat'执行将格式模板字符串拆分为字符数据和替换字段的工作。它根据需要调用'get_positional'和'get_index'方法（如下所述。）

Formatter 定义以下可覆盖方法：

```
-- get_value(key, args, kwargs)
-- check_unused_args(used_args, args, kwargs)
-- format_field(value, format_spec)
```

'get_value'用于检索给定的字段值。'key'参数是整数或字符串。如果是整数，它表示'args'中位置参数的索引；如果是字符串，则表示'kwargs'中的命名参数。

'args'参数设置为传递给'vformat'的位置参数列表，'kwargs'参数设置为位置参数的字典。

对于复合字段名称，这些函数仅针对字段名称的第一个组件调用；后续组件通过正常的属性和索引操作处理。

因此，例如，字段表达式'0.name'将导致用'key'参数 0 调用'get_value'。'name'属性将在'get_value'返回后通过调用内置'getattr'函数查找。

如果索引或关键字引用不存在的项，则应引发 IndexError/KeyError。

'check_unused_args'用于实现对未使用参数的检查（如果需要）。此函数的参数是格式字符串中实际引用的所有参数键的集合（位置参数为整数，命名参数为字符串），以及对传递给 vformat 的 args 和 kwargs 的引用。未使用的 args 集合可以从这些参数计算。'check_unused_args'假设如果检查失败则抛出异常。

'format_field'简单地调用全局'format'内置函数。提供此方法以便子类可以覆盖它。

为了更好地理解这些函数如何相互关联，下面是解释 vformat 一般操作的伪代码：

```python
def vformat(format_string, args, kwargs):

    # 输出缓冲区和使用过的 args 集合
    buffer = StringIO.StringIO()
    used_args = set()

    # 令牌是格式字段或字面字符串
    for token in self.parse(format_string):
        if is_format_field(token):
            # 将令牌拆分为字段值和格式说明符
            field_spec, _, format_spec = token.partition(":")

            # 检查显式类型转换
            explicit, _, field_spec = field_spec.rpartition("!")

            # 'first_part'是第一个'.'或'['之前的部分
            # 假设'get_first_part'返回 int 或 string，取决于语法。
            first_part = get_first_part(field_spec)
            value = self.get_value(first_part, args, kwargs)

            # 记录我们使用了此参数
            used_args.add(first_part)

            # 处理 [subfield] 或.subfield。假设'components'
            # 返回各种子字段的迭代器，不包括第一部分。
            for comp in components(field_spec):
                value = resolve_subfield(value, comp)

            # 处理显式类型转换
            if explicit == 'r':
                value = repr(value)
            elif explicit == 's':
                value = str(value)

            # 调用全局'format'函数并写出转换后的值。
            buffer.write(self.format_field(value, format_spec))

        else:
            buffer.write(token)

    self.check_unused_args(used_args, args, kwargs)
    return buffer.getvalue()
```

注意，Formatter 类的实际算法（将在 C 中实现）可能不是这里提出的那个。（实际的实现甚至可能不是一个'类'——vformat 可能只是调用一个接受其他可覆盖方法作为参数的 C 函数。）此代码示例的主要目的是说明可覆盖方法的调用顺序。

### 自定义 Formatter

本节描述了一些典型的 Formatter 对象自定义方式。

为了支持替代格式字符串语法，可以覆盖'vformat'方法来改变格式字符串的解析方式。

一个常见的愿望是支持'default'命名空间，这样您就不需要向 format() 方法传递关键字参数，而是可以使用预存在命名空间中的值。这可以通过如下覆盖 get_value() 轻松完成：

```python
class NamespaceFormatter(Formatter):
    def __init__(self, namespace={}):
        Formatter.__init__(self)
        self.namespace = namespace

    def get_value(self, key, args, kwds):
        if isinstance(key, str):
            try:
                # 首先检查显式传递的参数
                return kwds[key]
            except KeyError:
                return self.namespace[key]
        else:
            Formatter.get_value(key, args, kwds)
```

可以使用它来轻松创建允许访问全局变量的格式化函数，例如：

```python
fmt = NamespaceFormatter(globals())

greeting = "hello"
print(fmt.format("{greeting}, world!"))
```

可以使用 locals() 字典执行类似的技术来访问 locals 字典。

还可以创建'smart'命名空间格式化器，可以通过窥探调用堆栈自动访问 locals 和 globals。由于需要与不同版本的 Python 兼容，此类功能不会包含在标准库中，但预计有人会创建并发布执行此操作的配方。

另一种自定义类型是通过覆盖'format_field'方法来改变内置类型的格式化方式。（对于非内置类型，您可以简单地在该类型上定义__format__特殊方法。）因此，例如，您可以覆盖数字的格式化以在需要时输出科学记数法。

### 错误处理

格式化过程中可能发生两类异常：格式化器代码本身生成的异常和用户代码生成的异常（如字段对象的'getattr'函数）。

一般来说，格式化器代码本身生成的异常是"ValueError"类型——格式字符串的实际"值"中有错误。（这不总是真的；例如，string.format() 函数可能被传递非字符串作为第一个参数，这将导致 TypeError。）

与这些内部生成的 ValueError 异常关联的文本将指示格式字符串内的异常位置以及异常的性质。

对于用户代码生成的异常，将在跟踪堆栈中添加跟踪记录和虚拟帧以帮助确定字符串中发生异常的位置。插入的跟踪将指示错误发生在：

```
File "<format_string>;", line XX, in column_YY
```

其中 XX 和 YY 分别表示字符串中的行和字符位置信息。

## 替代语法

自然地，最有争议的问题之一是格式字符串的语法，特别是用于指示字段的标记约定。

我不试图详尽列出各种提议，我将介绍最广泛使用的。

- Shell 变量语法：`$name` 和`$(name)`（或在某些变体中，`${name}`）。这可能是最古老的约定，Perl 和其他许多人使用。当不使用花括号时，变量的长度通过词法扫描直到找到无效字符来确定。
此方案通常用于插值是隐式的情况——也就是说，在任何字符串可以包含插值变量且无需调用特殊替换函数的环境中。在这种情况下，重要的是防止插值行为意外发生，因此'$'（否则是一个相对不常用的字符）用于指示何时应发生行为。

然而，作者认为，在格式化显式调用的情况下，需要较少的注意来防止意外插值，在这种情况下可以使用更轻、不太笨重的语法。

- printf 及其表亲（'%'），包括添加字段索引的变体，以便字段可以按顺序插值。

- 其他仅括号变体。各种 MUD（多人地下城）如 MUSH 使用括号（例如 [name]）进行字符串插值。Microsoft .Net 库使用花括号（{}），以及与此提议非常相似的语法，尽管格式说明符的语法非常不同。[4]

- 反引号。这种方法的好处是最小化的语法杂乱，但它缺乏函数调用语法的许多好处（如复杂表达式参数、自定义格式化器等）。

- 其他变体包括 Ruby 的#{}, PHP 的{$name}等。

语法的某些具体方面需要额外评论：

1) 用于转义的反斜杠字符。此 PEP 的原始版本使用反斜杠而不是加倍来转义括号。这有效是因为 Python 字符串字面量中不符合标准反斜杠序列（如\n）的反斜杠保持不变。然而，这导致了一定程度的混淆，并导致了多重递归转义的情况，即\\\\{在括号前放置字面反斜杠。

2) 使用冒号字符（':'）作为格式说明符的分隔符。选择这个只是因为那是.Net 使用的。

## 替代功能提议

限制属性访问：PEP 的早期版本限制访问以下划线开头的属性的能力，例如"{0}._private"。然而，这是在调试时有用的能力，因此该功能被删除。

一些开发人员建议完全删除执行'getattr'和'getitem'访问的能力。然而，这与另一组开发人员的需要相冲突，他们强烈游说能够将大型 dict 作为单个参数传入（不使用**kwargs 语法将其展平为单个关键字参数），然后让格式字符串单独引用 dict 条目。

还有人建议扩展格式字符串中允许的表达式集。然而，这被认为违背了 TOOWTDI 的精神，因为在大多数情况下，可以通过在参数传递给格式化函数之前执行相同表达式来实现相同的效果。对于格式字符串用于在数据丰富的环境中执行任意格式化的情况，建议使用专门为此目的设计的模板引擎，如 Genshi [5] 或 Cheetah [6]。

许多其他功能被考虑和拒绝，因为它们可以通过子类化 Formatter 轻松实现，而不是将功能构建到基础实现中。这包括替代语法、格式字符串中的注释等。

## 安全考虑

历史上，字符串格式化一直是基于 Web 应用程序中安全漏洞的常见来源，特别是如果字符串格式化系统允许在格式字符串中嵌入任意表达式。

以不创建潜在安全漏洞的方式使用字符串格式化的最佳方法是永远不使用来自不可信源的格式字符串。

除此之外，下一个最好的方法是确保字符串格式化没有副作用。由于 Python 的开放性质，不可能保证任何非平凡操作具有此属性。本 PEP 所做的是将格式字符串中的表达式类型限制为可见副作用既罕见又被 Python 开发者文化强烈不鼓励的类型。因此，例如，允许属性访问，因为编写代码其中仅仅访问属性就有可见副作用将被认为是病态的（代码是否有不可见的副作用——如创建缓存项以加快查找——是无关紧要的。）
