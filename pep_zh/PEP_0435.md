# PEP 435 – 向 Python 标准库添加 Enum 类型

**Metadata:**

- **Author**: Barry Warsaw <barry&#32;&#97;t&#32;python.org>, Eli Bendersky <eliben&#32;&#97;t&#32;gmail.com>, Ethan Furman <ethan&#32;&#97;t&#32;stoneleaf.us>
- **Created**: 23-Feb-2013
- **Python-Version**: 3.4
- **Post-History**: 23-Feb-2013, 02-May-2013

## 摘要

本 PEP 提议向 Python 标准库添加一个枚举类型。

枚举是一组绑定到唯一、常量值的符号名称。在枚举内，值可以通过身份进行比较，并且可以遍历枚举本身。

## 讨论状态

向 Python 添加枚举类型的想法并不新鲜 - PEP 354 是 2005 年被拒绝的先前尝试。最近，在 python-ideas 邮件列表中启动了新一轮讨论 [3]。在几个线程中提出了许多新想法；经过 lengthy 讨论后，Guido 提议将 flufl.enum 添加到标准库 [4]。在 PyCon 2013 语言峰会上进一步讨论了这个问题。很明显，许多开发者希望看到一个继承自 int 的枚举，这可以让我们用具有友好字符串表示的枚举来替换标准库中的许多整数常量，而不会失去向后兼容性。几位感兴趣的核心开发者之间的进一步讨论导致了将 IntEnum 作为 Enum 的特例的提议。

Enum 和 IntEnum 之间的关键分歧在于与整数的比较是否具有语义意义。对于大多数枚举的用途来说，拒绝与整数比较是一个特性；与整数比较的枚举会通过传递性导致不相关类型的枚举之间的比较，这在大多数情况下是不希望的。然而，对于某些用途，希望与整数有更大的互操作性。例如，这是用枚举替换现有标准库常量（如 socket.AF_INET）的情况。

2013 年 4 月下旬的进一步讨论导致结论：枚举成员应属于其枚举的类型：type(Color.red) == Color。Guido 已就此问题发表决定 [5]，以及相关的不允许子类化枚举 [6]，除非它们没有定义任何枚举成员 [7]。

该 PEP 于 2013 年 5 月 10 日经 Guido 接受 [1]。

## 动机

[部分基于 PEP 354 中所述的动机]

枚举的特性对于定义不可变的、相关的常量值集非常有用，这些值可能具有也可能不具有语义意义。经典的例子是一周中的日子（星期日到星期六）和学校评估等级（'A' 到 'D'，以及 'F'）。其他例子包括错误状态值和定义过程中的状态。

可以简单地定义一些其他基本类型（如 int 或 str）的值序列来表示离散任意值。但是，枚举确保这些值与其他任何值都不同，特别是其他枚举中的值，并且没有意义的操作（"周三乘以二"）未针对这些值定义。它还提供了枚举值的方便的可打印表示，而不需要在定义它们时进行繁琐的重复（即没有 GREEN = 'green'）。

## 模块和类型名称

我们提议向标准库添加一个名为 enum 的模块。此模块公开的主要类型是 Enum。因此，要导入 Enum 类型，用户代码将运行：

```python
>>> from enum import Enum
```

## 新枚举类型的建议语义

### 创建 Enum

枚举使用类语法创建，这使得它们易于阅读和编写。另一种创建方法在功能 API 中描述。要定义枚举，请按如下方式子类化 Enum：

```python
>>> from enum import Enum
>>> class Color(Enum):
...     red = 1
...     green = 2
...     blue = 3
```

关于术语的说明：我们将 Color 称为枚举（或 enum），而 Color.red、Color.green 是枚举成员（或 enum members）。枚举成员也有值（Color.red 的值是 1 等）。

枚举成员具有人类可读的字符串表示：

```python
>>> print(Color.red)
Color.red
```

……而它们的 repr 包含更多信息：

```python
>>> print(repr(Color.red))
<Color.red: 1>
```

枚举成员的类型是它所属的枚举：

```python
>>> type(Color.red)
<Enum 'Color'>
>>> isinstance(Color.green, Color)
True
>>>
```

枚举还有一个包含其项名称的属性：

```python
>>> print(Color.red.name)
red
```

枚举支持按定义顺序迭代：

```python
>>> class Shake(Enum):
...     vanilla = 7
...     chocolate = 4
...     cookies = 9
...     mint = 3
...     
>>> for shake in Shake:
...     print(shake)
...     
Shake.vanilla
Shake.chocolate
Shake.cookies
Shake.mint
```

枚举成员是可哈希的，因此可以在字典和集合中使用：

```python
>>> apples = {}
>>> apples[Color.red] = 'red delicious'
>>> apples[Color.green] = 'granny smith'
>>> apples
{<Color.red: 1>: 'red delicious', <Color.green: 2>: 'granny smith'}
```

### 对枚举成员的程序化访问

有时需要以编程方式访问枚举中的成员（即 Color.red 不够用的情况，因为确切的颜色在编写程序时未知）。Enum 允许这样的访问：

```python
>>> Color(1)
<Color.red: 1>
>>> Color(3)
<Color.blue: 3>
```

如果你想按名称访问枚举成员，使用项访问：

```python
>>> Color['red']
<Color.red: 1>
>>> Color['green']
<Color.green: 2>
```

### 重复枚举成员和值

具有相同名称的两个枚举成员是无效的：

```python
>>> class Shape(Enum):
...     square = 2
...     square = 3
...
Traceback (most recent call last):
...
TypeError: Attempted to reuse key: square
```

但是，两个枚举成员可以具有相同的值。给定两个具有相同值的成员 A 和 B（A 先定义），B 是 A 的别名。A 和 B 的值的按值查找将返回 A。B 的按名称查找也将返回 A：

```python
>>> class Shape(Enum):
...     square = 2
...     diamond = 1
...     circle = 3
...     alias_for_square = 2
...
>>> Shape.square
<Shape.square: 2>
>>> Shape.alias_for_square
<Shape.square: 2>
>>> Shape(2)
<Shape.square: 2>
```

遍历枚举的成员不提供别名：

```python
>>> list(Shape)
[<Shape.square: 2>, <Shape.diamond: 1>, <Shape.circle: 3>]
```

特殊属性 `__members__` 是一个将名称映射到成员的有序字典。它包括枚举中定义的所有名称，包括别名：

```python
>>> for name, member in Shape.__members__.items():
...     name, member
...
('square', <Shape.square: 2>)
('diamond', <Shape.diamond: 1>)
('circle', <Shape.circle: 3>)
('alias_for_square', <Shape.square: 2>)
```

`__members__` 属性可用于对枚举成员进行详细的程序化访问。例如，查找所有别名：

```python
>>> [name for name, member in Shape.__members__.items() if member.name != name]
['alias_for_square']
```

### 比较

枚举成员按身份进行比较：

```python
>>> Color.red is Color.red
True
>>> Color.red is Color.blue
False
>>> Color.red is not Color.blue
True
```

不支持枚举值之间的有序比较。枚举不是整数（但见下文 IntEnum）：

```python
>>> Color.red < Color.blue
Traceback (most recent call last):
 File "<stdin>", line 1, in <module>
TypeError: unorderable types: Color() < Color()
```

但定义了相等比较：

```python
>>> Color.blue == Color.red
False
>>> Color.blue != Color.red
True
>>> Color.blue == Color.blue
True
```

与非枚举值/不同类型枚举的比较将永远比较为不相等（同样，IntEnum 被明确设计为表现不同，见下文）：


```python
>>> Color.blue == 2
False
```

### 允许的枚举成员和属性

上面的例子使用整数作为枚举值。使用整数简短而方便（并由功能 API 默认提供），但并非严格强制。在绝大多数用例中，人们不关心枚举的实际值是什么。但如果值很重要，枚举可以有任意值。

枚举是 Python 类，可以像往常一样具有方法和特殊方法。如果我们有这个枚举：

```python
class Mood(Enum):
    funky = 1
    happy = 3

    def describe(self):
        # self 是这里的成员
        return self.name, self.value

    def __str__(self):
        return 'my custom str! {0}'.format(self.value)

    @classmethod
    def favorite_mood(cls):
        # cls 这里是枚举
        return cls.happy
```

然后：

```python
>>> Mood.favorite_mood()
<Mood.happy: 3>
>>> Mood.happy.describe()
('happy', 3)
>>> str(Mood.funky)
'my custom str! 1'
```

允许的规则如下：在枚举内定义的所有属性将成为此枚举的成员，`__dunder__` 名称和描述符 [9] 除外；方法也是描述符。

### 受限的枚举子类化

只有当枚举没有定义任何成员时才允许子类化枚举。因此这是禁止的：

```python
>>> class MoreColor(Color):
...     pink = 17
...
TypeError: Cannot extend enumerations
```

但这是允许的：

```python
>>> class Foo(Enum):
...     def some_behavior(self):
...         pass
...
>>> class Bar(Foo):
...     happy = 1
...     sad = 2
...
```

这个决定的理由由 Guido 在 [6] 中给出。允许子类化定义成员的枚举将导致一些重要的类型和实例不变量被违反。另一方面，在枚举组之间共享一些常见行为是有意义的，子类化空枚举也用于实现 IntEnum。

### IntEnum

提议了 Enum 的一个变体，它也是 int 的子类。IntEnum 的成员可以与整数比较；通过扩展，不同类型的整数枚举也可以相互比较：

```python
>>> from enum import IntEnum
>>> class Shape(IntEnum):
...     circle = 1
...     square = 2
...
>>> class Request(IntEnum):
...     post = 1
...     get = 2
...
>>> Shape == 1
False
>>> Shape.circle == 1
True
>>> Shape.circle == Request.post
True
```

但是它们仍然不能与 Enum 比较：

```python
>>> class Shape(IntEnum):
...     circle = 1
...     square = 2
...
>>> class Color(Enum):
...     red = 1
...     green = 2
...
>>> Shape.circle == Color.red
False
```

IntEnum 值在其他方面表现得像整数：

```python
>>> int(Shape.circle)
1
>>> ['a', 'b', 'c'][Shape.circle]
'b'
>>> [i for i in range(Shape.square)]
[0, 1]
```

对于绝大多数代码，强烈推荐使用 Enum，因为 IntEnum 破坏了枚举的一些语义承诺（通过与整数可比较，从而通过传递性与不相关的枚举可比较）。它应该仅在特殊情况下使用，没有其他选择时；例如，当整数常量被枚举替换并且需要与仍然期望整数的代码向后兼容时。

### 其他派生枚举

IntEnum 将是 enum 模块的一部分。但是，独立实现它非常简单：

```python
class IntEnum(int, Enum):
    pass
```

这展示了如何定义类似的派生枚举，例如一个混合 str 而不是 int 的 StrEnum。

一些规则：

- 子类化 Enum 时，混入类型必须出现在基类序列中 Enum 之前，如上文的 IntEnum 示例。

- 虽然 Enum 可以有任意类型的成员，但一旦你混入额外的类型，所有成员必须具有该类型的值，例如上面的 int。此限制不适用于仅添加方法而不指定其他数据类型（如 int 或 str）的混入。

### Pickling

枚举可以被 pickle 和 unpickle：

```python
>>> from enum.tests.fruit import Fruit
>>> from pickle import dumps, loads
>>> Fruit.tomato is loads(dumps(Fruit.tomato))
True
```
Pickle 的通常限制适用：可 pickle 的枚举必须定义在模块的顶层，因为 unpickle 需要从该模块导入它们。

### 功能 API

Enum 类是可调用的，提供以下功能 API：

```python
>>> Animal = Enum('Animal', 'ant bee cat dog')
>>> Animal
<Enum 'Animal'>
>>> Animal.ant
<Animal.ant: 1>
>>> Animal.ant.value
1
>>> list(Animal)
[<Animal.ant: 1>, <Animal.bee: 2>, <Animal.cat: 3>, <Animal.dog: 4>]
```
此 API 的语义类似于 namedtuple。调用 Enum 的第一个参数是枚举的名称。使用功能 API 创建的枚举的 pickle 将在 CPython 和 PyPy 上工作，但对于 IronPython 和 Jython，你可能需要明确指定模块名称，如下所示：

```python
>>> Animals = Enum('Animals', 'ant bee cat dog', module=__name__)
```

第二个参数是枚举成员名称的来源。它可以是名称的空格分隔字符串、名称序列、带键/值对的 2 元组序列，或名称到值的映射（例如字典）。最后两个选项允许为枚举分配任意值；其他选项自动分配从 1 开始的递增整数。返回从 Enum 派生的新类。换句话说，上面的 Animal 赋值等同于：

```python
>>> class Animals(Enum):
...     ant = 1
...     bee = 2
...     cat = 3
...     dog = 4
```

默认值为 1 作为起始数字而不是 0 的原因是 0 在布尔意义上是 False，但枚举成员都求值为 True。

## 建议的变体

在邮件列表的讨论期间提出了一些变体。这里是一些更受欢迎的。

### flufl.enum

flufl.enum 是本 PEP 最初基于的参考实现。最终，决定不纳入 flufl.enum，因为它的设计将枚举成员与枚举分离，所以前者不是后者的实例。它的设计还明确允许子类化枚举以用更多成员扩展它们（由于成员/枚举分离，在这种方案下 flufl.enum 中的类型不变量不会被违反）。

### 不必为枚举指定值

Michael Foord 提议（Tim Delaney 提供了概念验证实现）使用元类魔术使这成为可能：

```python
class Color(Enum):
    red, green, blue
```

值实际上只在第一次查找时分配。

优点：更简洁的语法，对于非常常见的任务（只需列出枚举名称而不关心值）需要更少的输入。

缺点：实现中涉及很多魔术，使得首次看到时甚至对这种枚举的定义也令人困惑。此外，明确优于隐式。

### 使用特殊名称或形式自动分配枚举值

避免指定枚举值的另一种方法是使用特殊名称或形式自动分配它们。例如：

```python
class Color(Enum):
    red = None  # auto-assigned to 0
    green = None  # auto-assigned to 1
    blue = None  # auto-assigned to 2
```

更灵活：

```python
class Color(Enum):
    red = 7
    green = None  # auto-assigned to 8
    blue = 19
    purple = None  # auto-assigned to 20
```

此主题的一些变体：

- 从 enum 包自动导入的特殊名称。

- Georg Brandl 提议使用省略号 (...) 而不是 None 来达到相同的效果。

优点：不需要手动输入值。使更改和扩展枚举更容易，特别是对于大型枚举。

缺点：在许多简单情况下实际上需要更长的输入。明确与隐式的论点也适用于此。

## 在标准库中的用例

Python 标准库有许多地方使用枚举来替换当前用于表示它们的其他惯用法将是有益的。此类用途可分为两类：面向用户代码的常量和内部常量。

面向用户代码的常量，如 os.SEEK_*、socket 模块常量、decimal 舍入模式和 HTML 错误代码，可能需要向后兼容性，因为用户代码可能期望整数。如上所述的 IntEnum 提供了所需的语义；作为 int 的子类，它不会影响期望整数的用户代码，而另一方面允许枚举值的可打印表示：

```python
>>> import socket
>>> family = socket.AF_INET
>>> family == 2
True
>>> print(family)
SocketFamily.AF_INET
```

内部常量不被用户代码看到，但由 stdlib 模块内部使用。这些可以用 Enum 实现。通过非常部分浏览 stdlib 发现的一些例子：binhex、imaplib、http/client、urllib/robotparser、idlelib、concurrent.futures、turtledemo。

此外，查看 Twisted 库的代码，有许多用例可以用枚举替换内部状态常量。对于很多网络代码（特别是协议实现）也可以这样说，在使用 Tulip 库编写的测试协议中也可以看到。
