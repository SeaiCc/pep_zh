# PEP 404 – Python 2.8 非发布计划

**元数据：**

- **作者**: Barry Warsaw <barry at python.org>
- **创建时间**: 09-Nov-2011
- **Python 版本**: 2.8

## 摘要

本文档描述了 Python 2.8 的非开发和非发布计划。

## 非发布计划

当前非计划如下：

- 2.8 最终版：永远不会

## 官方声明

规则六：没有官方的 Python 2.8 版本。永远不会有官方的 Python 2.8 版本。它是一个"非发布"。Python 2.7 是 Python 2 开发系列的终点。

## 升级路径

从 Python 2.7 的官方升级路径是迁移到 Python 3。

## 现在来点完全不同的东西

说正经的，没有官方 Python 2.8 版本有重要的原因，你应该计划迁移到 Python 3。

Python（截至本文撰写时）已经超过 20 年历史，Guido 和社区在这些年间学到了很多。Guido 关于 Python 3 的最初概念是对语言进行更改，主要是去除前面版本中积累的问题。Python 3 不是完全重新设计，而是语言的演变，虽然明确排除了与 Python 2 的完全向后兼容，但也不接受随意的语法或语义更改。在大多数情况下，Python 2 代码可以相对容易地转换为 Python 3，有时完全通过 2to3 等工具机械地完成（还有一个非微不足道的语言子集可以在 2.7 和 3.x 上无需修改运行）。

由于维护多个版本的 Python 对 Python 开发者的资源来说是重大负担，而且由于 Python 3 体现的语言和库改进如此重要，因此决定以 Python 2.7 结束 Python 2 系列。因此，所有新开发都发生在 Python 3 开发系列中，永远不会有官方的 Python 2.8 版本。然而，Python 2.7 将被维护比通常更长的时间。

以下是 Python 3 中重大改进的一些亮点。你可以阅读更多关于 Python 2 和 Python 3 之间差异的详细信息。还有很多关于从 Python 2 移植到 Python 3 的优秀指南。

### 字符串和字节

Python 2 的基本原始字符串称为 8 位字符串，它们在 Python 2 中扮演双重角色，既是 ASCII 文本又是字节序列。虽然 Python 2 也有 unicode 字符串类型，但核心字符串类型的基本模糊性，加上 Python 2 在组合时支持从 8 位字符串自动强制转换为 unicode 对象的默认行为，经常导致 UnicodeErrors。Python 3 的标准字符串类型基于 Unicode，Python 3 添加了专用的 bytes 类型，但关键是，不提供 bytes 和 unicode 字符串之间的自动强制转换。语言接近隐式强制转换的是几个基于文本的 API，如果没有明确指定编码，它们假设默认编码（通常是 UTF-8）。因此，核心解释器、其 I/O 库、模块名称等在区分 unicode 字符串和 bytes 方面是清晰的。Python 3 的 unicode 支持甚至扩展到文件系统，因此原生支持非 ASCII 文件名。

这种字符串/字节的清晰度通常是过渡现有代码到 Python 3 的困难来源，因为许多第三方库和应用本身在这个区分上是模糊的。但一旦迁移，大多数 UnicodeErrors 可以被消除。

### 数字

Python 2 有两种基本整数类型，原生机器大小的 int 类型和任意长度的 long 类型。这些在 Python 3 中已合并为单个 int 类型，类似于 Python 2 的 long 类型。

此外，整数除法现在对非整数结果产生浮点数。

### 类

Python 2 有两个核心类层次结构，通常称为经典类和新式类。后者允许诸如从内置基本类型继承、支持基于描述符的工具如 property 内置函数，并为处理多重继承提供一般更合理和连贯的系统。Python 3 提供了完全放弃对经典类支持的机会，所以 Python 3 中的所有类自动使用新式语义（虽然现在这是个误称）。没有必要显式地从 object 继承或设置默认元类型来启用它们（事实上，在模块级别设置默认元类型不再支持——默认元类型总是 object）。

显式指定元类的机制也已更改，使用类头部行中的 metaclass 关键字参数，而不是类体中的__metaclass__魔法属性。

### 多种拼写

在 Python 2 中有许多情况下存在某些构造的多种拼写，例如 repr() 和反引号，或两个不等运算符 != 和 <>。在所有情况下，Python 3 都选择了恰好一种拼写并移除了另一种（例如保留了 repr() 和 !=）。

### 导入

在 Python 3 中，包内的隐式相对导入不再可用——只支持绝对导入和显式相对导入。此外，星号导入（例如 from x import *）仅允许在模块级代码中。

此外，标准库的一些区域已重新组织，使命名方案更直观。一些很少使用的内置函数已 relocated 到标准库模块中。

### 迭代器和视图

许多在 Python 2 中返回具体列表的 API，在 Python 3 中现在返回迭代器或轻量级视图。
